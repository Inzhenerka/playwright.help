### headerValue {#response-header-value}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.15</font><x-search>response.headerValue</x-search>

Возвращает значение заголовка, соответствующего имени. Имя нечувствительно к регистру. Если несколько заголовков имеют одно и то же имя (кроме `set-cookie`), они возвращаются в виде списка, разделенного `, `. Для `set-cookie` используется разделитель `\n`. Если заголовки не найдены, возвращается `null`.

**Использование**

```java
Response.headerValue(name);
```

**Аргументы**
- `name` [String]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-header-value-option-name"/><a href="#response-header-value-option-name" class="list-anchor">#</a>
  
  Имя заголовка.

**Возвращает**
- [null] | [String]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-header-value-return"/><a href="#response-header-value-return" class="list-anchor">#</a>

### headers {#response-headers}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено до v1.9</font><x-search>response.headers</x-search>

Объект с HTTP заголовками ответа. Имена заголовков приведены к нижнему регистру. Обратите внимание, что этот метод не возвращает заголовки, связанные с безопасностью, включая заголовки, связанные с cookie. Вы можете использовать [Response.allHeaders()](/api/class-response.mdx#response-all-headers) для полного списка заголовков, включающего информацию о `cookie`.

**Использование**

```java
Response.headers();
```

**Возвращает**
- [Map]&lt;[String], [String]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-headers-return"/><a href="#response-headers-return" class="list-anchor">#</a>

### headersArray {#response-headers-array}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.15</font><x-search>response.headersArray</x-search>

Массив со всеми HTTP заголовками запроса, связанными с этим ответом. В отличие от [Response.allHeaders()](/api/class-response.mdx#response-all-headers), имена заголовков НЕ приведены к нижнему регистру. Заголовки с несколькими записями, такие как `Set-Cookie`, появляются в массиве несколько раз.

**Использование**

```java
Response.headersArray();
```

**Возвращает**
- [List]&lt;`HttpHeader`&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-headers-array-return"/><a href="#response-headers-array-return" class="list-anchor">#</a>
  - `name` [String]
    
    Имя заголовка.
  - `value` [String]
    
    Значение заголовка.

### securityDetails {#response-security-details}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.13</font><x-search>response.securityDetails</x-search>

Возвращает информацию о SSL и других аспектах безопасности.

**Использование**

```java
Response.securityDetails();
```

**Возвращает**
- [null] | SecurityDetails<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-security-details-return"/><a href="#response-security-details-return" class="list-anchor">#</a>
  - `issuer` [String] *(опционально)*
    
    Компонент Common Name из поля Issuer сертификата. Это следует использовать только в информационных целях. Опционально.
  - `protocol` [String] *(опционально)*
    
    Конкретный используемый протокол TLS (например, `TLS 1.3`). Опционально.
  - `subjectName` [String] *(опционально)*
    
    Компонент Common Name из поля Subject сертификата. Это следует использовать только в информационных целях. Опционально.
  - `validFrom` [double] *(опционально)*
    
    Unix-временная метка (в секундах), указывающая, когда сертификат становится действительным. Опционально.
  - `validTo` [double] *(опционально)*
    
    Unix-временная метка (в секундах), указывающая, когда сертификат становится недействительным. Опционально.
