Когда контекст браузера создается с опцией `recordVideo`, каждая страница имеет связанный с ней объект видео.

```java
System.out.println(page.video().path());
```

---

## Методы {#methods}

### delete {#video-delete}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.11</font><x-search>video.delete</x-search>

Удаляет видеофайл. При необходимости будет ожидать завершения видео.

**Использование**

```java
Video.delete();
```

**Возвращает**
- [void]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="video-delete-return"/><a href="#video-delete-return" class="list-anchor">#</a>

---

### path {#video-path}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено до v1.9</font><x-search>video.path</x-search>

Возвращает путь в файловой системе, куда будет записано это видео. Гарантируется, что видео будет записано в файловую систему после закрытия контекста браузера. Этот метод вызывает ошибку при удаленном подключении.

**Использование**

```java
Video.path();
```

**Возвращает**
- [Path]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="video-path-return"/><a href="#video-path-return" class="list-anchor">#</a>

---

### saveAs {#video-save-as}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.11</font><x-search>video.saveAs</x-search>

Сохраняет видео в указанное пользователем место. Этот метод можно безопасно вызывать, пока видео еще записывается, или после закрытия страницы. Этот метод ждет, пока страница не будет закрыта и видео полностью сохранено.

**Использование**

```java
Video.saveAs(path);
```

**Аргументы**
- `path` [Path]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="video-save-as-option-path"/><a href="#video-save-as-option-path" class="list-anchor">#</a>
  
  Путь, по которому должно быть сохранено видео.

**Возвращает**
- [void]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="video-save-as-return"/><a href="#video-save-as-return" class="list-anchor">#</a>
