### header_value {#response-header-value}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.15</font><x-search>response.header_value</x-search>

Возвращает значение заголовка, соответствующего имени. Имя не чувствительно к регистру. Если несколько заголовков имеют одно и то же имя (кроме `set-cookie`), они возвращаются в виде списка, разделенного `, `. Для `set-cookie` используется разделитель `\n`. Если заголовки не найдены, возвращается `null`.

**Использование**

```python
response.header_value(name)
```

**Аргументы**
- `name` [str]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-header-value-option-name"/><a href="#response-header-value-option-name" class="list-anchor">#</a>
  
  Имя заголовка.

**Возвращает**
- [NoneType] | [str]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-header-value-return"/><a href="#response-header-value-return" class="list-anchor">#</a>

### headers_array {#response-headers-array}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.15</font><x-search>response.headers_array</x-search>

Массив со всеми HTTP-заголовками запроса, связанными с этим ответом. В отличие от [response.all_headers()](/api/class-response.mdx#response-all-headers), имена заголовков НЕ приводятся к нижнему регистру. Заголовки с несколькими записями, такие как `Set-Cookie`, появляются в массиве несколько раз.

**Использование**

```python
response.headers_array()
```

**Возвращает**
- [List]\[[Dict]\]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-headers-array-return"/><a href="#response-headers-array-return" class="list-anchor">#</a>
  - `name` [str]
    
    Имя заголовка.
  - `value` [str]
    
    Значение заголовка.

### security_details {#response-security-details}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.13</font><x-search>response.security_details</x-search>

Возвращает информацию о SSL и других аспектах безопасности.

**Использование**

```python
response.security_details()
```

**Возвращает**
- [NoneType] | [Dict]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-security-details-return"/><a href="#response-security-details-return" class="list-anchor">#</a>
  - `issuer` [str] *(опционально)*
    
    Компонент Common Name из поля Issuer сертификата. Должен использоваться только в информационных целях. Опционально.
  - `protocol` [str] *(опционально)*
    
    Конкретный используемый протокол TLS (например, `TLS 1.3`). Опционально.
  - `subjectName` [str] *(опционально)*
    
    Компонент Common Name из поля Subject сертификата. Должен использоваться только в информационных целях. Опционально.
  - `validFrom` [float] *(опционально)*
    
    Unix-временная метка (в секундах), указывающая, когда сертификат становится действительным. Опционально.
  - `validTo` [float] *(опционально)*
    
    Unix-временная метка (в секундах), указывающая, когда сертификат становится недействительным. Опционально.
