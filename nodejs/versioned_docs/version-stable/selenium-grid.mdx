## Введение

Playwright может подключаться к [Selenium Grid Hub](https://www.selenium.dev/documentation/grid/), который работает на Selenium 4, для запуска браузера **Google Chrome** или **Microsoft Edge**, вместо запуска браузера на локальной машине. Обратите внимание, что эта функция является **экспериментальной** и имеет соответствующий приоритет.

:::warning

Существует риск, что интеграция Playwright с Selenium Grid Hub может перестать работать в будущем. Убедитесь, что вы взвесили риски и преимущества перед использованием этой функции.

Внутри Playwright подключается к браузеру с помощью веб-сокета [Chrome DevTools Protocol](https://chromedevtools.github.io/devtools-protocol/). В настоящее время Selenium 4 предоставляет эту возможность. Однако это [может измениться в будущем](https://github.com/SeleniumHQ/selenium/issues/11590#issuecomment-1436113950). Если Selenium уберет эту возможность, Playwright перестанет работать с ним.

Перед подключением Playwright к вашему Selenium Grid убедитесь, что грид работает с [Selenium WebDriver](https://www.selenium.dev/documentation/webdriver/). Например, запустите [один из примеров](https://github.com/SeleniumHQ/selenium/tree/trunk/javascript/node/selenium-webdriver/example) и передайте переменную окружения `SELENIUM_REMOTE_URL`. Если пример webdriver не работает, проверьте наличие ошибок в выводе вашего Selenium hub/node/standalone и ищите [проблемы Selenium](https://github.com/SeleniumHQ/selenium/issues) для возможного решения.

## Запуск Selenium Grid

Если вы запускаете распределенный Selenium Grid, Playwright требует, чтобы узлы selenium были зарегистрированы с доступным адресом, чтобы он мог подключаться к браузерам. Чтобы убедиться, что это работает как ожидалось, установите переменную окружения `SE_NODE_GRID_URL`, указывающую на хаб при запуске узлов selenium.

```bash
# Запуск узла selenium
SE_NODE_GRID_URL="http://<selenium-hub-ip>:4444" java -jar selenium-server-<version>.jar node
```

## Подключение Playwright к Selenium Grid

Чтобы подключить Playwright к **Selenium Grid 4**, установите переменную окружения `SELENIUM_REMOTE_URL`, указывающую на ваш Selenium Grid Hub. Обратите внимание, что это работает только для Google Chrome и Microsoft Edge.

```bash
SELENIUM_REMOTE_URL=http://<selenium-hub-ip>:4444 npx playwright test
```

Вам не нужно изменять ваш код, просто используйте ваш тестовый фреймворк или [browserType.launch()](/api/class-browsertype.mdx#browser-type-launch) как обычно.

### Передача дополнительных возможностей

Если ваш грид требует установки дополнительных возможностей (например, вы используете внешнюю службу), вы можете установить переменную окружения `SELENIUM_REMOTE_CAPABILITIES`, чтобы предоставить возможности в формате JSON.

```bash
SELENIUM_REMOTE_URL=http://<selenium-hub-ip>:4444 SELENIUM_REMOTE_CAPABILITIES="{'mygrid:options':{os:'windows',username:'John',password:'secure'}}" npx playwright test
```

### Передача дополнительных заголовков

Если ваш грид требует установки дополнительных заголовков (например, вам нужно предоставить токен авторизации для использования браузеров в вашем облаке), вы можете установить переменную окружения `SELENIUM_REMOTE_HEADERS`, чтобы предоставить заголовки в формате JSON.

```bash
SELENIUM_REMOTE_URL=http://<selenium-hub-ip>:4444 SELENIUM_REMOTE_HEADERS="{'Authorization':'Basic b64enc'}" npx playwright test
```

### Подробные логи

Запустите с переменной окружения `DEBUG=pw:browser*`, чтобы увидеть, как Playwright подключается к Selenium Grid.

```bash
DEBUG=pw:browser* SELENIUM_REMOTE_URL=http://internal.grid:4444 npx playwright test
```

Если вы создаете проблему, пожалуйста, включите этот лог.

## Использование Selenium Docker

Один из простых способов использовать Selenium Grid — запустить официальные контейнеры docker. Узнайте больше в документации [selenium docker images](https://github.com/SeleniumHQ/docker-selenium). Для конвенции тегирования образов [узнайте больше](https://github.com/SeleniumHQ/docker-selenium/wiki/Tagging-Convention#selenium-grid-4x-and-above).

### Режим Standalone

Вот пример запуска selenium в режиме standalone и подключения Playwright к нему. Обратите внимание, что хаб и узел находятся на одном `localhost`, и мы передаем переменную окружения `SE_NODE_GRID_URL`, указывающую на него.

Сначала запустите Selenium.

```bash
docker run -d -p 4444:4444 --shm-size="2g" -e SE_NODE_GRID_URL="http://localhost:4444" selenium/standalone-chromium:latest
```

Затем запустите Playwright.

```bash
SELENIUM_REMOTE_URL=http://localhost:4444 npx playwright test
```

### Режим Hub и узлы

Вот пример запуска хаба selenium и одного узла selenium, а также подключения Playwright к хабу. Обратите внимание, что у хаба и узла разные IP-адреса, и мы передаем переменную окружения `SE_NODE_GRID_URL`, указывающую на хаб, при запуске контейнеров узлов.

Сначала запустите контейнер хаба и один или несколько контейнеров узлов.

```bash
docker run -d -p 4442-4444:4442-4444 --name selenium-hub selenium/hub:4.25.0
docker run -d -p 5555:5555 \
    --shm-size="2g" \
    -e SE_EVENT_BUS_HOST=<selenium-hub-ip> \
    -e SE_EVENT_BUS_PUBLISH_PORT=4442 \
    -e SE_EVENT_BUS_SUBSCRIBE_PORT=4443 \
    -e SE_NODE_GRID_URL="http://<selenium-hub-ip>:4444"
    selenium/node-chromium:4.25.0
```

Затем запустите Playwright.

```bash
SELENIUM_REMOTE_URL=http://<selenium-hub-ip>:4444 npx playwright test
```

## Selenium 3

Внутри Playwright подключается к браузеру с помощью веб-сокета [Chrome DevTools Protocol](https://chromedevtools.github.io/devtools-protocol/). Selenium 4 предоставляет эту возможность, в то время как Selenium 3 — нет.

Это означает, что Selenium 3 поддерживается в режиме "лучших усилий", когда Playwright пытается подключиться к узлу грида напрямую. Узлы грида должны быть непосредственно доступны с машины, на которой работает Playwright.
