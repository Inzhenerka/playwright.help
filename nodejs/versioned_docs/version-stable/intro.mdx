## Введение

Playwright Test был создан специально для удовлетворения потребностей тестирования "конец в конец". Playwright поддерживает все современные движки рендеринга, включая Chromium, WebKit и Firefox. Тестируйте на Windows, Linux и macOS, локально или в CI, в безголовом или обычном режиме с нативной мобильной эмуляцией Google Chrome для Android и Mobile Safari.

## Установка Playwright

Начните с установки Playwright с помощью npm, yarn или pnpm. В качестве альтернативы вы также можете начать и запустить свои тесты, используя [расширение VS Code](./getting-started-vscode.mdx).

<Tabs defaultValue="npm" values={[ {label: 'npm', value: 'npm'}, {label: 'yarn', value: 'yarn'}, {label: 'pnpm', value: 'pnpm'} ] }>

<TabItem value="npm">

```bash
npm init playwright@latest
```

</TabItem>

<TabItem value="yarn">

```bash
yarn create playwright
```

</TabItem>

<TabItem value="pnpm">

```bash
pnpm create playwright
```

</TabItem>

Запустите команду установки и выберите следующее, чтобы начать:
- Выберите между TypeScript или JavaScript (по умолчанию используется TypeScript)
- Название вашей папки с тестами (по умолчанию tests или e2e, если у вас уже есть папка с тестами в проекте)
- Добавить рабочий процесс GitHub Actions для легкого запуска тестов в CI
- Установить браузеры Playwright (по умолчанию true)

## Что установлено

Playwright загрузит необходимые браузеры, а также создаст следующие файлы.

```bash
playwright.config.ts
package.json
package-lock.json
tests/
  example.spec.ts
tests-examples/
  demo-todo-app.spec.ts
```

[playwright.config](./test-configuration.mdx) — это место, где вы можете добавить конфигурацию для Playwright, включая изменение браузеров, на которых вы хотите запускать Playwright. Если вы запускаете тесты внутри уже существующего проекта, зависимости будут добавлены непосредственно в ваш `package.json`.

Папка `tests` содержит базовый пример теста, чтобы помочь вам начать тестирование. Для более детального примера ознакомьтесь с папкой `tests-examples`, которая содержит тесты, написанные для тестирования приложения todo.

## Запуск примера теста

По умолчанию тесты будут запускаться на всех 3 браузерах: Chromium, Firefox и WebKit, используя 3 рабочих процесса. Это можно настроить в [файле playwright.config](./test-configuration.mdx). Тесты запускаются в безголовом режиме, что означает, что браузер не откроется при запуске тестов. Результаты тестов и логи тестов будут отображаться в терминале.

## HTML отчеты о тестах

После завершения вашего теста будет сгенерирован [HTML репортер](./test-reporters.mdx#html-reporter), который покажет вам полный отчет о ваших тестах, позволяя фильтровать отчет по браузерам, пройденным тестам, неудачным тестам, пропущенным тестам и ненадежным тестам. Вы можете щелкнуть на каждом тесте и изучить ошибки теста, а также каждый шаг теста. По умолчанию HTML отчет открывается автоматически, если некоторые тесты не прошли.

## Обновление Playwright

Чтобы обновить Playwright до последней версии, выполните следующую команду:

<Tabs defaultValue="npm" values={[ {label: 'npm', value: 'npm'}, {label: 'yarn', value: 'yarn'}, {label: 'pnpm', value: 'pnpm'} ] }>

<TabItem value="npm">

```bash
npm install -D @playwright/test@latest
# Также загрузите новые бинарные файлы браузеров и их зависимости:
npx playwright install --with-deps
```

</TabItem>

<TabItem value="yarn">

```bash
yarn add --dev @playwright/test@latest
# Также загрузите новые бинарные файлы браузеров и их зависимости:
yarn playwright install --with-deps
```

</TabItem>

<TabItem value="pnpm">

```bash
pnpm install --save-dev @playwright/test@latest
# Также загрузите новые бинарные файлы браузеров и их зависимости:
pnpm exec playwright install --with-deps
```

</TabItem>

</Tabs>

Вы всегда можете проверить, какая версия Playwright у вас установлена, выполнив следующую команду:

## Системные требования
- Последняя версия Node.js 18, 20 или 22.
- Windows 10+, Windows Server 2016+ или Windows Subsystem for Linux (WSL).
- macOS 13 Ventura или более поздняя версия.
- Debian 12, Ubuntu 22.04, Ubuntu 24.04 на архитектуре x86-64 и arm64.
