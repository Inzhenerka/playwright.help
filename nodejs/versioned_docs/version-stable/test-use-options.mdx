## Введение

Помимо настройки тестового раннера, вы также можете настроить [Эмуляцию](#emulation-options), [Сеть](#network-options) и [Запись](#recording-options) для [Браузера] или [Контекста браузера]. Эти параметры передаются в объект `use: {}` в конфигурации Playwright.

### Основные параметры

Установите базовый URL и состояние хранилища для всех тестов:

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    // Базовый URL для использования в действиях, таких как `await page.goto('/')`.
    baseURL: 'http://localhost:3000',

    // Заполняет контекст заданным состоянием хранилища.
    storageState: 'state.json',
  },
});
```

| Параметр | Описание |
| :- | :- |
| [testOptions.baseURL](/api/class-testoptions.mdx#test-options-base-url) | Базовый URL, используемый для всех страниц в контексте. Позволяет навигировать, используя только путь, например `page.goto('/settings')`. |
| [testOptions.storageState](/api/class-testoptions.mdx#test-options-storage-state) | Заполняет контекст заданным состоянием хранилища. Полезно для простой аутентификации, [узнайте больше](./auth.mdx). |

### Параметры эмуляции

С помощью Playwright вы можете эмулировать реальное устройство, такое как мобильный телефон или планшет. См. наш [гид по проектам](./test-projects.mdx) для получения дополнительной информации о эмуляции устройств. Вы также можете эмулировать `"геолокацию"`, `"локаль"` и `"часовой пояс"` для всех тестов или для конкретного теста, а также установить `"разрешения"` для отображения уведомлений или изменения `"цветовой схемы"`. См. наш [гид по эмуляции](./emulation.mdx), чтобы узнать больше.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    // Эмулирует медиа-функцию `'prefers-colors-scheme'`.
    colorScheme: 'dark',

    // Геолокация контекста.
    geolocation: { longitude: 12.492507, latitude: 41.889938 },

    // Эмулирует локаль пользователя.
    locale: 'en-GB',

    // Предоставляет указанные разрешения контексту браузера.
    permissions: ['geolocation'],

    // Эмулирует часовой пояс пользователя.
    timezoneId: 'Europe/Paris',

    // Размер окна для всех страниц в контексте.
    viewport: { width: 1280, height: 720 },
  },
});
```

| Параметр | Описание |
| :- | :- |
| [testOptions.colorScheme](/api/class-testoptions.mdx#test-options-color-scheme) | [Эмулирует](./emulation.mdx#color-scheme-and-media) медиа-функцию `'prefers-colors-scheme'`, поддерживаемые значения: `'light'` и `'dark'` |
| [testOptions.geolocation](/api/class-testoptions.mdx#test-options-geolocation) | [Геолокация](./emulation.mdx#geolocation) контекста. |
| [testOptions.locale](/api/class-testoptions.mdx#test-options-locale) | [Эмулирует](./emulation.mdx#locale--timezone) локаль пользователя, например `en-GB`, `de-DE` и т.д. |
| [testOptions.permissions](/api/class-testoptions.mdx#test-options-permissions) | Список [разрешений](./emulation.mdx#permissions), предоставляемых всем страницам в контексте. |
| [testOptions.timezoneId](/api/class-testoptions.mdx#test-options-timezone-id) | Изменяет [часовой пояс](./emulation.mdx#locale--timezone) контекста. |
| [testOptions.viewport](/api/class-testoptions.mdx#test-options-viewport) | [Размер окна](./emulation.mdx#viewport), используемый для всех страниц в контексте. |

### Параметры сети

Доступные параметры для настройки сетевого взаимодействия:

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    // Нужно ли автоматически загружать все вложения.
    acceptDownloads: false,

    // Объект, содержащий дополнительные HTTP-заголовки, которые будут отправлены с каждым запросом.
    extraHTTPHeaders: {
      'X-My-Header': 'value',
    },

    // Учетные данные для HTTP-аутентификации.
    httpCredentials: {
      username: 'user',
      password: 'pass',
    },

    // Нужно ли игнорировать ошибки HTTPS во время навигации.
    ignoreHTTPSErrors: true,

    // Нужно ли эмулировать отключение сети.
    offline: true,

    // Настройки прокси, используемые для всех страниц в тесте.
    proxy: {
      server: 'http://myproxy.com:3128',
      bypass: 'localhost',
    },
  },
});
```

| Параметр | Описание |
| :- | :- |
| [testOptions.acceptDownloads](/api/class-testoptions.mdx#test-options-accept-downloads) | Нужно ли автоматически загружать все вложения, по умолчанию `true`. [Узнайте больше](./downloads.mdx) о работе с загрузками. |
| [testOptions.extraHTTPHeaders](/api/class-testoptions.mdx#test-options-extra-http-headers) | Объект, содержащий дополнительные HTTP-заголовки, которые будут отправлены с каждым запросом. Все значения заголовков должны быть строками. |
| [testOptions.httpCredentials](/api/class-testoptions.mdx#test-options-http-credentials) | Учетные данные для [HTTP-аутентификации](./network.mdx#http-authentication). |
| [testOptions.ignoreHTTPSErrors](/api/class-testoptions.mdx#test-options-ignore-https-errors) | Нужно ли игнорировать ошибки HTTPS во время навигации. |
| [testOptions.offline](/api/class-testoptions.mdx#test-options-offline) | Нужно ли эмулировать отключение сети. |
| [testOptions.proxy](/api/class-testoptions.mdx#test-options-proxy) | [Настройки прокси](./network.mdx#http-proxy), используемые для всех страниц в тесте. |

:::note

Вам не нужно настраивать ничего, чтобы имитировать сетевые запросы. Просто определите пользовательский [Маршрут], который имитирует сеть для контекста браузера. См. наш [гид по имитации сети](./network.mdx), чтобы узнать больше.
:::

### Параметры записи

С помощью Playwright вы можете захватывать скриншоты, записывать видео, а также трассировки вашего теста. По умолчанию они отключены, но вы можете включить их, установив параметры `screenshot`, `video` и `trace` в вашем файле `playwright.config.js`.

Файлы трассировки, скриншоты и видео будут появляться в каталоге вывода тестов, обычно `test-results`.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    // Захватывать скриншот после каждого сбоя теста.
    screenshot: 'only-on-failure',

    // Записывать трассу только при повторной попытке теста в первый раз.
    trace: 'on-first-retry',

    // Записывать видео только при повторной попытке теста в первый раз.
    video: 'on-first-retry'
  },
});
```

| Параметр | Описание |
| :- | :- |
| [testOptions.screenshot](/api/class-testoptions.mdx#test-options-screenshot) | Захватывать [скриншоты](./screenshots.mdx) вашего теста. Опции включают `'off'`, `'on'` и `'only-on-failure'` |
| [testOptions.trace](/api/class-testoptions.mdx#test-options-trace) | Playwright может создавать трассы тестов во время выполнения тестов. Позже вы можете просмотреть трассу и получить подробную информацию о выполнении Playwright, открыв [Trace Viewer](./trace-viewer.mdx). Опции включают: `'off'`, `'on'`, `'retain-on-failure'` и `'on-first-retry'`  |
| [testOptions.video](/api/class-testoptions.mdx#test-options-video) | Playwright может записывать [видео](./videos.mdx) для ваших тестов. Опции включают: `'off'`, `'on'`, `'retain-on-failure'` и `'on-first-retry'` |

### Другие параметры

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    // Максимальное время, которое может занять каждое действие, такое как `click()`. По умолчанию 0 (без ограничения).
    actionTimeout: 0,

    // Название браузера, который запускает тесты. Например `chromium`, `firefox`, `webkit`.
    browserName: 'chromium',

    // Переключает обход Content-Security-Policy.
    bypassCSP: true,

    // Канал для использования, например "chrome", "chrome-beta", "msedge", "msedge-beta".
    channel: 'chrome',

    // Запускать браузер в безголовом режиме.
    headless: false,

    // Изменить атрибут по умолчанию data-testid.
    testIdAttribute: 'pw-test-id',
  },
});
```

| Параметр | Описание |
| :- | :- |
| [testOptions.actionTimeout](/api/class-testoptions.mdx#test-options-action-timeout) | Таймаут для каждого действия Playwright в миллисекундах. По умолчанию `0` (без таймаута). Узнайте больше о [таймаутах](./test-timeouts.mdx) и о том, как установить их для одного теста. |
| [testOptions.browserName](/api/class-testoptions.mdx#test-options-browser-name) | Название браузера, который запускает тесты. По умолчанию 'chromium'. Опции включают `chromium`, `firefox` или `webkit`. |
| [testOptions.bypassCSP](/api/class-testoptions.mdx#test-options-bypass-csp) | Переключает обход Content-Security-Policy. Полезно, когда CSP включает производственный источник. По умолчанию `false`. |
| [testOptions.channel](/api/class-testoptions.mdx#test-options-channel) | Канал браузера для использования. [Узнайте больше](./browsers.mdx) о различных браузерах и каналах. |
| [testOptions.headless](/api/class-testoptions.mdx#test-options-headless) | Нужно ли запускать браузер в безголовом режиме, что означает, что браузер не отображается при выполнении тестов. По умолчанию `true`. |
| [testOptions.testIdAttribute](/api/class-testoptions.mdx#test-options-test-id-attribute) | Изменяет атрибут [`data-testid`](./locators.mdx#locate-by-test-id), используемый локаторами Playwright. |

### Дополнительные параметры браузера и контекста

Любые параметры, принимаемые [browserType.launch()](/api/class-browsertype.mdx#browser-type-launch) или [browser.newContext()](/api/class-browser.mdx#browser-new-context), могут быть помещены в `launchOptions` или `contextOptions` соответственно в разделе `use`.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    launchOptions: {
      slowMo: 50,
    },
  },
});
```

Тем не менее, наиболее распространенные параметры, такие как `headless` или `viewport`, доступны непосредственно в разделе `use` - см. [основные параметры](#basic-options), [эмуляцию](#emulation-options) или [сеть](#network-options).

### Явное создание контекста и наследование параметров

Если вы используете встроенный фикстуру `browser`, вызов [browser.newContext()](/api/class-browser.mdx#browser-new-context) создаст контекст с параметрами, унаследованными от конфигурации:

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    userAgent: 'some custom ua',
    viewport: { width: 100, height: 100 },
  },
});
```

Пример теста, иллюстрирующий, что начальные параметры контекста установлены:

```js
test('должен наследовать параметры использования в контексте при использовании встроенной фикстуры браузера', async ({
  browser,
}) => {
  const context = await browser.newContext();
  const page = await context.newPage();
  expect(await page.evaluate(() => navigator.userAgent)).toBe('some custom ua');
  expect(await page.evaluate(() => window.innerWidth)).toBe(100);
  await context.close();
});
```

### Области конфигурации

Вы можете настраивать Playwright глобально, для проекта или для теста. Например, вы можете установить локаль, которая будет использоваться глобально, добавив `locale` в параметр `use` конфигурации Playwright, а затем переопределить ее для конкретного проекта, используя параметр `project` в конфигурации. Вы также можете переопределить ее для конкретного теста, добавив `test.use({})` в файл теста и передав параметры.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    locale: 'en-GB'
  },
});
```

Вы можете переопределить параметры для конкретного проекта, используя параметр `project` в конфигурации Playwright.

```js
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  projects: [
    {
      name: 'chromium',
      use: {
        ...devices['Desktop Chrome'],
        locale: 'de-DE',
      },
    },
  ],
});
```

Вы можете переопределить параметры для конкретного файла теста, используя метод `test.use()` и передавая параметры. Например, чтобы запустить тесты с французской локалью для конкретного теста:

```js
import { test, expect } from '@playwright/test';

test.use({ locale: 'fr-FR' });

test('пример', async ({ page }) => {
  // ...
});
```

То же самое работает внутри блока describe. Например, чтобы запустить тесты в блоке describe с французской локалью:

```js
import { test, expect } from '@playwright/test';

test.describe('блок французского языка', () => {

  test.use({ locale: 'fr-FR' });

  test('пример', async ({ page }) => {
    // ...
  });
});
```
