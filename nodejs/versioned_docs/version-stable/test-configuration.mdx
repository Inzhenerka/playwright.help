## Введение

Playwright предлагает множество опций для настройки выполнения ваших тестов. Вы можете указать эти опции в файле конфигурации. Обратите внимание, что опции тестового раннера являются **верхнего уровня**, не помещайте их в секцию `use`.

## Основная конфигурация

Вот некоторые из самых распространенных опций конфигурации.

```js
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  // Ищите тестовые файлы в директории "tests", относительно этого файла конфигурации.
  testDir: 'tests',

  // Запускайте все тесты параллельно.
  fullyParallel: true,

  // Провалите сборку на CI, если вы случайно оставили test.only в исходном коде.
  forbidOnly: !!process.env.CI,

  // Повторная попытка только на CI.
  retries: process.env.CI ? 2 : 0,

  // Отказ от параллельных тестов на CI.
  workers: process.env.CI ? 1 : undefined,

  // Репортер для использования
  reporter: 'html',

  use: {
    // Базовый URL для использования в действиях, таких как `await page.goto('/')`.
    baseURL: 'http://localhost:3000',

    // Собирать трассировку при повторной попытке неудачного теста.
    trace: 'on-first-retry',
  },
  // Настройка проектов для основных браузеров.
  projects: [
    {
      name: 'chromium',
      use: { ...devices['Desktop Chrome'] },
    },
  ],
  // Запустите ваш локальный сервер разработки перед началом тестов.
  webServer: {
    command: 'npm run start',
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI,
  },
});
```

| Опция | Описание |
| :- | :- |
| [testConfig.forbidOnly](/api/class-testconfig.mdx#test-config-forbid-only) | Указывает, следует ли завершать с ошибкой, если какие-либо тесты помечены как `test.only`. Полезно на CI.|
| [testConfig.fullyParallel](/api/class-testconfig.mdx#test-config-fully-parallel) | Запуск всех тестов во всех файлах параллельно. См. [Параллелизм](./test-parallel) и [Шардинг](./test-sharding) для получения дополнительных сведений. |
| [testConfig.projects](/api/class-testconfig.mdx#test-config-projects) | Запуск тестов в нескольких конфигурациях или на нескольких браузерах |
| [testConfig.reporter](/api/class-testconfig.mdx#test-config-reporter) | Репортер для использования. См. [Тестовые репортеры](/test-reporters.mdx), чтобы узнать больше о доступных репортерах. |
| [testConfig.retries](/api/class-testconfig.mdx#test-config-retries) | Максимальное количество попыток повторного запуска для каждого теста. См. [Повторные попытки тестов](/test-retries.mdx), чтобы узнать больше о повторных попытках.|
| [testConfig.testDir](/api/class-testconfig.mdx#test-config-test-dir) | Директория с тестовыми файлами. |
| [testConfig.use](/api/class-testconfig.mdx#test-config-use)  | Опции с `use{}` |
| [testConfig.webServer](/api/class-testconfig.mdx#test-config-web-server) | Для запуска сервера во время тестов используйте опцию `webServer` |
| [testConfig.workers](/api/class-testconfig.mdx#test-config-workers) | Максимальное количество параллельных рабочих процессов для параллелизации тестов. Также может быть установлено как процент логических ядер ЦП, например, `'50%'`. См. [Параллелизм](./test-parallel) и [Шардинг](./test-sharding) для получения дополнительных сведений. |

## Фильтрация тестов

Фильтруйте тесты по шаблонам glob или регулярным выражениям.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  // Шаблоны glob или регулярные выражения для игнорирования тестовых файлов.
  testIgnore: '*test-assets',

  // Шаблоны glob или регулярные выражения, которые соответствуют тестовым файлам.
  testMatch: '*todo-tests/*.spec.ts',
});
```

| Опция | Описание |
| :- | :- |
| [testConfig.testIgnore](/api/class-testconfig.mdx#test-config-test-ignore) | Шаблоны glob или регулярные выражения, которые должны быть проигнорированы при поиске тестовых файлов. Например, `'*test-assets'` |
| [testConfig.testMatch](/api/class-testconfig.mdx#test-config-test-match) | Шаблоны glob или регулярные выражения, которые соответствуют тестовым файлам. Например, `'*todo-tests/*.spec.ts'`. По умолчанию Playwright запускает файлы <code>.*(test&#124;spec)\.(js&#124;ts&#124;mjs)</code>. |

## Расширенная конфигурация

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  // Папка для артефактов тестов, таких как скриншоты, видео, трассировки и т. д.
  outputDir: 'test-results',

  // путь к файлам глобальной настройки.
  globalSetup: require.resolve('./global-setup'),

  // путь к файлам глобального завершения.
  globalTeardown: require.resolve('./global-teardown'),

  // Каждому тесту отводится 30 секунд.
  timeout: 30000,

});
```

| Опция | Описание |
| :- | :- |
| [testConfig.globalSetup](/api/class-testconfig.mdx#test-config-global-setup) | Путь к файлу глобальной настройки. Этот файл будет загружен и выполнен перед всеми тестами. Он должен экспортировать одну функцию. |
| [testConfig.globalTeardown](/api/class-testconfig.mdx#test-config-global-teardown) | Путь к файлу глобального завершения. Этот файл будет загружен и выполнен после всех тестов. Он должен экспортировать одну функцию. |
| [testConfig.outputDir](/api/class-testconfig.mdx#test-config-output-dir) | Папка для артефактов тестов, таких как скриншоты, видео, трассировки и т. д. |
| [testConfig.timeout](/api/class-testconfig.mdx#test-config-timeout) | Playwright устанавливает [тайм-аут](./test-timeouts.mdx) для каждого теста, по умолчанию 30 секунд. Время, затраченное функцией теста, тестовыми фикстурами и хуками beforeEach, включается в тайм-аут теста. |

## Опции Expect

Конфигурация для библиотеки утверждений expect.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  expect: {
    // Максимальное время, которое expect() должен ждать, чтобы условие было выполнено.
    timeout: 5000,

    toHaveScreenshot: {
      // Допустимое количество пикселей, которые могут отличаться, по умолчанию не установлено.
      maxDiffPixels: 10,
    },

    toMatchSnapshot: {
      // Допустимое соотношение пикселей, которые отличаются от общего количества пикселей, между 0 и 1.
      maxDiffPixelRatio: 0.1,
    },
  },

});
```

| Опция | Описание |
| :- | :- |
| [testConfig.expect](/api/class-testconfig.mdx#test-config-expect) | [Утверждения с приоритетом веба](./test-assertions.mdx), такие как `expect(locator).toHaveText()`, имеют отдельный тайм-аут в 5 секунд по умолчанию. Это максимальное время, которое `expect()` должен ждать, чтобы условие было выполнено. Узнайте больше о [тайм-аутах тестов и expect](./test-timeouts.mdx) и о том, как установить их для одного теста. |
| [expect(page).toHaveScreenshot()](/api/class-pageassertions.mdx#page-assertions-to-have-screenshot-1) | Конфигурация для метода `expect(locator).toHaveScreenshot()`. |
| [expect(value).toMatchSnapshot()](/api/class-snapshotassertions.mdx#snapshot-assertions-to-match-snapshot-1)| Конфигурация для метода `expect(locator).toMatchSnapshot()`. |
