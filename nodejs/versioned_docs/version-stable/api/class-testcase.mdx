`TestCase` соответствует каждому вызову [test()](/api/class-test.mdx#test-call) в тестовом файле. Когда один [test()](/api/class-test.mdx#test-call) выполняется в нескольких проектах или повторяется несколько раз, он будет иметь несколько объектов `TestCase` в соответствующих наборах проектов.


---

## Методы

### ok {#test-case-ok}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.10</font><x-search>testCase.ok</x-search>

Определяет, считается ли тест выполняемым корректно. Некорректные тесты завершают выполнение с ненулевым кодом выхода.

**Использование**

```js
testCase.ok();
```

**Возвращает**
- [boolean]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-case-ok-return"/><a href="#test-case-ok-return" class="list-anchor">#</a>

---

### outcome {#test-case-outcome}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.10</font><x-search>testCase.outcome</x-search>

Результат тестирования для этого теста. Обратите внимание, что результат не совпадает с [testResult.status](/api/class-testresult.mdx#test-result-status):
* Тест, который ожидается как неудачный и действительно неудачен, имеет статус `'expected'`.
* Тест, который проходит при повторной попытке, имеет статус `'flaky'`.

**Использование**

```js
testCase.outcome();
```

**Возвращает**
- "skipped" | "expected" | "unexpected" | "flaky"<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-case-outcome-return"/><a href="#test-case-outcome-return" class="list-anchor">#</a>

## Свойства

### annotations {#test-case-annotations}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.10</font><x-search>testCase.annotations</x-search>

Список аннотаций, применимых к текущему тесту. Включает:
* аннотации, определенные в тесте или наборе через [test()](/api/class-test.mdx#test-call) и [test.describe()](/api/class-test.mdx#test-describe);
* аннотации, неявно добавленные методами [test.skip()](/api/class-test.mdx#test-skip), [test.fixme()](/api/class-test.mdx#test-fixme) и [test.fail()](/api/class-test.mdx#test-fail);
* аннотации, добавленные к [testInfo.annotations](/api/class-testinfo.mdx#test-info-annotations) во время выполнения теста.

Аннотации доступны во время выполнения теста через [testInfo.annotations](/api/class-testinfo.mdx#test-info-annotations).

Узнайте больше о [тестовых аннотациях](../test-annotations.mdx).

**Использование**

```js
testCase.annotations
```

**Тип**
- [Array]&lt;[Object]&gt;
  - `type` [string]
    
    Тип аннотации, например `'skip'` или `'fail'`.
  - `description` [string] *(необязательно)*
    
    Необязательное описание.

### expectedStatus {#test-case-expected-status}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.10</font><x-search>testCase.expectedStatus</x-search>

Ожидаемый статус теста.
* Тесты, помеченные как [test.skip()](/api/class-test.mdx#test-skip) или [test.fixme()](/api/class-test.mdx#test-fixme), ожидаются как `'skipped'`.
* Тесты, помеченные как [test.fail()](/api/class-test.mdx#test-fail), ожидаются как `'failed'`.
* Другие тесты ожидаются как `'passed'`.

Смотрите также [testResult.status](/api/class-testresult.mdx#test-result-status) для фактического статуса.

**Использование**

```js
testCase.expectedStatus
```

**Тип**
- "passed" | "failed" | "timedOut" | "skipped" | "interrupted"
