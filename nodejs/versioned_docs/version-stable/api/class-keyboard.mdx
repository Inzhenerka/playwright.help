Класс Keyboard предоставляет API для управления виртуальной клавиатурой. Высокоуровневый API — это [keyboard.type()](/api/class-keyboard.mdx#keyboard-type), который принимает необработанные символы и генерирует соответствующие события `keydown`, `keypress`/`input` и `keyup` на вашей странице.

Для более тонкого управления вы можете использовать [keyboard.down()](/api/class-keyboard.mdx#keyboard-down), [keyboard.up()](/api/class-keyboard.mdx#keyboard-up) и [keyboard.insertText()](/api/class-keyboard.mdx#keyboard-insert-text), чтобы вручную вызывать события, как если бы они были сгенерированы с реальной клавиатуры.

Пример удерживания `Shift` для выбора и удаления текста:

```js
await page.keyboard.type('Hello World!');
await page.keyboard.press('ArrowLeft');

await page.keyboard.down('Shift');
for (let i = 0; i < ' World'.length; i++)
  await page.keyboard.press('ArrowLeft');
await page.keyboard.up('Shift');

await page.keyboard.press('Backspace');
// Результирующий текст будет 'Hello!'
```

Пример нажатия заглавной `A`:

```js
await page.keyboard.press('Shift+KeyA');
// или
await page.keyboard.press('Shift+A');
```

Пример для вызова выбора всего с помощью клавиатуры:

```js
await page.keyboard.press('ControlOrMeta+A');
```

## Методы

### down {#keyboard-down}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено до v1.9</font><x-search>keyboard.down</x-search>

Отправляет событие `keydown`.

[key](/api/class-keyboard.mdx#keyboard-down-option-key) может указывать на предполагаемое значение [keyboardEvent.key](https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/key) или один символ для генерации текста. Расширенный набор значений [key](/api/class-keyboard.mdx#keyboard-down-option-key) можно найти [здесь](https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/key/Key_Values). Примеры клавиш:

`F1` - `F12`, `Digit0`- `Digit9`, `KeyA`- `KeyZ`, `Backquote`, `Minus`, `Equal`, `Backslash`, `Backspace`, `Tab`, `Delete`, `Escape`, `ArrowDown`, `End`, `Enter`, `Home`, `Insert`, `PageDown`, `PageUp`, `ArrowRight`, `ArrowUp` и т.д.

Также поддерживаются следующие модификаторы: `Shift`, `Control`, `Alt`, `Meta`, `ShiftLeft`, `ControlOrMeta`. `ControlOrMeta` разрешается в `Control` на Windows и Linux и в `Meta` на macOS.

Удерживание `Shift` будет вводить текст, соответствующий [key](/api/class-keyboard.mdx#keyboard-down-option-key), в верхнем регистре.

Если [key](/api/class-keyboard.mdx#keyboard-down-option-key) — это один символ, он чувствителен к регистру, поэтому значения `a` и `A` будут генерировать разные тексты.

Если [key](/api/class-keyboard.mdx#keyboard-down-option-key) — это клавиша-модификатор, `Shift`, `Meta`, `Control` или `Alt`, последующие нажатия клавиш будут отправлены с активным модификатором. Чтобы отпустить клавишу-модификатор, используйте [keyboard.up()](/api/class-keyboard.mdx#keyboard-up).

После нажатия клавиши один раз, последующие вызовы [keyboard.down()](/api/class-keyboard.mdx#keyboard-down) будут иметь [repeat](https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/repeat), установленный в true. Чтобы отпустить клавишу, используйте [keyboard.up()](/api/class-keyboard.mdx#keyboard-up).

:::note

Клавиши-модификаторы действительно влияют на `keyboard.down`. Удерживание `Shift` будет вводить текст в верхнем регистре.
:::

**Использование**

```js
await keyboard.down(key);
```

**Аргументы**
- `key` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="keyboard-down-option-key"/><a href="#keyboard-down-option-key" class="list-anchor">#</a>
  
  Название клавиши для нажатия или символ для генерации, например, `ArrowLeft` или `a`.

**Возвращает**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="keyboard-down-return"/><a href="#keyboard-down-return" class="list-anchor">#</a>

### insertText {#keyboard-insert-text}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено до v1.9</font><x-search>keyboard.insertText</x-search>

Отправляет только событие `input`, не генерирует события `keydown`, `keyup` или `keypress`.

**Использование**

```js
page.keyboard.insertText('嗨');
```

:::note
Клавиши-модификаторы НЕ влияют на `keyboard.insertText`. Удерживание `Shift` не будет вводить текст в верхнем регистре.
:::

**Аргументы**
- `text` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="keyboard-insert-text-option-text"/><a href="#keyboard-insert-text-option-text" class="list-anchor">#</a>
  
  Устанавливает ввод на указанное текстовое значение.

**Возвращает**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="keyboard-insert-text-return"/><a href="#keyboard-insert-text-return" class="list-anchor">#</a>

### press {#keyboard-press}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено до v1.9</font><x-search>keyboard.press</x-search>

:::tip

В большинстве случаев вам следует использовать [locator.press()](/api/class-locator.mdx#locator-press) вместо этого.
:::

[key](/api/class-keyboard.mdx#keyboard-press-option-key) может указывать на предполагаемое значение [keyboardEvent.key](https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/key) или один символ для генерации текста. Расширенный набор значений [key](/api/class-keyboard.mdx#keyboard-press-option-key) можно найти [здесь](https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/key/Key_Values). Примеры клавиш:

`F1` - `F12`, `Digit0`- `Digit9`, `KeyA`- `KeyZ`, `Backquote`, `Minus`, `Equal`, `Backslash`, `Backspace`, `Tab`, `Delete`, `Escape`, `ArrowDown`, `End`, `Enter`, `Home`, `Insert`, `PageDown`, `PageUp`, `ArrowRight`, `ArrowUp` и т.д.

Также поддерживаются следующие модификаторы: `Shift`, `Control`, `Alt`, `Meta`, `ShiftLeft`, `ControlOrMeta`. `ControlOrMeta` разрешается в `Control` на Windows и Linux и в `Meta` на macOS.

Удерживание `Shift` будет вводить текст, соответствующий [key](/api/class-keyboard.mdx#keyboard-press-option-key), в верхнем регистре.

Если [key](/api/class-keyboard.mdx#keyboard-press-option-key) — это один символ, он чувствителен к регистру, поэтому значения `a` и `A` будут генерировать разные тексты.

Поддерживаются такие сочетания клавиш, как `key: "Control+o"`, `key: "Control++` или `key: "Control+Shift+T"`. Когда указано с модификатором, модификатор удерживается во время нажатия последующей клавиши.

**Использование**

```js
const page = await browser.newPage();
await page.goto('https://keycode.info');
await page.keyboard.press('A');
await page.screenshot({ path: 'A.png' });
await page.keyboard.press('ArrowLeft');
await page.screenshot({ path: 'ArrowLeft.png' });
await page.keyboard.press('Shift+O');
await page.screenshot({ path: 'O.png' });
await browser.close();
```

Сочетание для [keyboard.down()](/api/class-keyboard.mdx#keyboard-down) и [keyboard.up()](/api/class-keyboard.mdx#keyboard-up).

**Аргументы**
- `key` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="keyboard-press-option-key"/><a href="#keyboard-press-option-key" class="list-anchor">#</a>
  
  Название клавиши для нажатия или символ для генерации, например, `ArrowLeft` или `a`.
- `options` [Object] *(необязательно)*
  - `delay` [number] *(необязательно)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="keyboard-press-option-delay"/><a href="#keyboard-press-option-delay" class="list-anchor">#</a>
    
    Время ожидания между `keydown` и `keyup` в миллисекундах. По умолчанию 0.

**Возвращает**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="keyboard-press-return"/><a href="#keyboard-press-return" class="list-anchor">#</a>

### type {#keyboard-type}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено до v1.9</font><x-search>keyboard.type</x-search>

:::caution

В большинстве случаев вам следует использовать [locator.fill()](/api/class-locator.mdx#locator-fill) вместо этого. Вам нужно нажимать клавиши по одной, только если на странице есть специальная обработка клавиатуры — в этом случае используйте [locator.pressSequentially()](/api/class-locator.mdx#locator-press-sequentially).
:::

Отправляет события `keydown`, `keypress`/`input` и `keyup` для каждого символа в тексте.

Чтобы нажать специальную клавишу, такую как `Control` или `ArrowDown`, используйте [keyboard.press()](/api/class-keyboard.mdx#keyboard-press).

**Использование**

```js
await page.keyboard.type('Hello'); // Вводит мгновенно
await page.keyboard.type('World', { delay: 100 }); // Вводит медленнее, как пользователь
```

:::note
Клавиши-модификаторы НЕ влияют на `keyboard.type`. Удерживание `Shift` не будет вводить текст в верхнем регистре.
:::

:::note
Для символов, которые отсутствуют на американской клавиатуре, будет отправлено только событие `input`.
:::

**Аргументы**
- `text` [string]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="keyboard-type-option-text"/><a href="#keyboard-type-option-text" class="list-anchor">#</a>
  
  Текст для ввода в активный элемент.
- `options` [Object] *(необязательно)*
  - `delay` [number] *(необязательно)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="keyboard-type-option-delay"/><a href="#keyboard-type-option-delay" class="list-anchor">#</a>
    
    Время ожидания между нажатиями клавиш в миллисекундах. По умолчанию 0.

**Возвращает**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="keyboard-type-return"/><a href="#keyboard-type-return" class="list-anchor">#</a>
