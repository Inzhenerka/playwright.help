Когда сетевой маршрут настраивается с помощью [page.route()](/api/class-page.mdx#page-route) или [browserContext.route()](/api/class-browsercontext.mdx#browser-context-route), объект `Route` позволяет обрабатывать этот маршрут.

Узнайте больше о [сетевом взаимодействии](../network.mdx).


---

## Методы

### abort {#route-abort}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено до v1.9</font><x-search>route.abort</x-search>

Прерывает запрос маршрута.

**Использование**

```js
await route.abort();
await route.abort(errorCode);
```

**Аргументы**
- `errorCode` [string] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-abort-option-error-code"/><a href="#route-abort-option-error-code" class="list-anchor">#</a>
  
  Необязательный код ошибки. По умолчанию `failed`, может быть одним из следующих:
  * `'aborted'` - Операция была прервана (в результате действия пользователя)
  * `'accessdenied'` - Доступ к ресурсу, отличному от сети, был запрещен
  * `'addressunreachable'` - IP-адрес недоступен. Это обычно означает, что нет маршрута к указанному хосту или сети.
  * `'blockedbyclient'` - Клиент решил заблокировать запрос.
  * `'blockedbyresponse'` - Запрос не удался, потому что ответ был доставлен с требованиями, которые не были выполнены (например, проверки 'X-Frame-Options' и 'Content-Security-Policy').
  * `'connectionaborted'` - Соединение было прервано из-за отсутствия ACK для отправленных данных.
  * `'connectionclosed'` - Соединение было закрыто (соответствует TCP FIN).
  * `'connectionfailed'` - Попытка соединения не удалась.
  * `'connectionrefused'` - Попытка соединения была отклонена.
  * `'connectionreset'` - Соединение было сброшено (соответствует TCP RST).
  * `'internetdisconnected'` - Соединение с Интернетом было потеряно.
  * `'namenotresolved'` - Имя хоста не может быть разрешено.
  * `'timedout'` - Операция превысила время ожидания.
  * `'failed'` - Произошла общая ошибка.

**Возвращает**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-abort-return"/><a href="#route-abort-return" class="list-anchor">#</a>

### continue {#route-continue}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено до v1.9</font><x-search>route.continue</x-search>

Отправляет запрос маршрута в сеть с необязательными переопределениями.

**Использование**

```js
await page.route('**/*', async (route, request) => {
  // Переопределение заголовков
  const headers = {
    ...request.headers(),
    foo: 'foo-value', // установить заголовок "foo"
    bar: undefined, // удалить заголовок "bar"
  };
  await route.continue({ headers });
});
```

**Аргументы**
- `options` [Object] *(необязательный)*
  - `headers` [Object]&lt;[string], [string]&gt; *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-continue-option-headers"/><a href="#route-continue-option-headers" class="list-anchor">#</a>
    
    Если задано, изменяет HTTP-заголовки запроса. Значения заголовков будут преобразованы в строку.
  - `method` [string] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-continue-option-method"/><a href="#route-continue-option-method" class="list-anchor">#</a>
    
    Если задано, изменяет метод запроса (например, GET или POST).
  - `postData` [string] | [Buffer] | [Serializable] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-continue-option-post-data"/><a href="#route-continue-option-post-data" class="list-anchor">#</a>
    
    Если задано, изменяет данные поста запроса.
  - `url` [string] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-continue-option-url"/><a href="#route-continue-option-url" class="list-anchor">#</a>
    
    Если задано, изменяет URL запроса. Новый URL должен иметь тот же протокол, что и оригинальный.

**Возвращает**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-continue-return"/><a href="#route-continue-return" class="list-anchor">#</a>

**Подробности**

Опция [headers](/api/class-route.mdx#route-continue-option-headers) применяется как к маршрутизируемому запросу, так и к любым перенаправлениям, которые он инициирует. Однако [url](/api/class-route.mdx#route-continue-option-url), [method](/api/class-route.mdx#route-continue-option-method) и [postData](/api/class-route.mdx#route-continue-option-post-data) применяются только к оригинальному запросу и не передаются на перенаправленные запросы.

[route.continue()](/api/class-route.mdx#route-continue) немедленно отправит запрос в сеть, другие соответствующие обработчики не будут вызваны. Используйте [route.fallback()](/api/class-route.mdx#route-fallback), если вы хотите, чтобы следующий соответствующий обработчик в цепочке был вызван.

### fallback {#route-fallback}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.23</font><x-search>route.fallback</x-search>

Продолжает запрос маршрута с необязательными переопределениями. Метод аналогичен [route.continue()](/api/class-route.mdx#route-continue), с той разницей, что другие соответствующие обработчики будут вызваны перед отправкой запроса.

**Использование**

Когда несколько маршрутов соответствуют заданному шаблону, они выполняются в порядке, обратном их регистрации. Таким образом, последний зарегистрированный маршрут всегда может переопределить все предыдущие. В приведенном ниже примере запрос будет обрабатываться сначала самым нижним обработчиком, затем он вернется к предыдущему, и в конце будет прерван первым зарегистрированным маршрутом.

```js
await page.route('**/*', async route => {
  // Выполняется последним.
  await route.abort();
});
await page.route('**/*', async route => {
  // Выполняется вторым.
  await route.fallback();
});
await page.route('**/*', async route => {
  // Выполняется первым.
  await route.fallback();
});
```

Регистрация нескольких маршрутов полезна, когда вы хотите, чтобы отдельные обработчики обрабатывали разные виды запросов, например, API-вызовы против ресурсов страницы или GET-запросы против POST-запросов, как в приведенном ниже примере.

```js
// Обработка GET-запросов.
await page.route('**/*', async route => {
  if (route.request().method() !== 'GET') {
    await route.fallback();
    return;
  }
  // Обработка только GET.
  // ...
});

// Обработка POST-запросов.
await page.route('**/*', async route => {
  if (route.request().method() !== 'POST') {
    await route.fallback();
    return;
  }
  // Обработка только POST.
  // ...
});
```

Также можно изменить запрос, возвращаясь к последующему обработчику, таким образом промежуточный обработчик маршрута может изменить url, метод, заголовки и postData запроса.

```js
await page.route('**/*', async (route, request) => {
  // Переопределение заголовков
  const headers = {
    ...request.headers(),
    foo: 'foo-value', // установить заголовок "foo"
    bar: undefined, // удалить заголовок "bar"
  };
  await route.fallback({ headers });
});
```

Используйте [route.continue()](/api/class-route.mdx#route-continue), чтобы немедленно отправить запрос в сеть, в этом случае другие соответствующие обработчики не будут вызваны.

**Аргументы**
- `options` [Object] *(необязательный)*
  - `headers` [Object]&lt;[string], [string]&gt; *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fallback-option-headers"/><a href="#route-fallback-option-headers" class="list-anchor">#</a>
    
    Если задано, изменяет HTTP-заголовки запроса. Значения заголовков будут преобразованы в строку.
  - `method` [string] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fallback-option-method"/><a href="#route-fallback-option-method" class="list-anchor">#</a>
    
    Если задано, изменяет метод запроса (например, GET или POST).
  - `postData` [string] | [Buffer] | [Serializable] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fallback-option-post-data"/><a href="#route-fallback-option-post-data" class="list-anchor">#</a>
    
    Если задано, изменяет данные поста запроса.
  - `url` [string] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fallback-option-url"/><a href="#route-fallback-option-url" class="list-anchor">#</a>
    
    Если задано, изменяет URL запроса. Новый URL должен иметь тот же протокол, что и оригинальный. Изменение URL не повлияет на соответствие маршрута, все маршруты соответствуют исходному URL запроса.

**Возвращает**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fallback-return"/><a href="#route-fallback-return" class="list-anchor">#</a>

### fetch {#route-fetch}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.29</font><x-search>route.fetch</x-search>

Выполняет запрос и получает результат, не выполняя его, чтобы ответ мог быть изменен и затем выполнен.

**Использование**

```js
await page.route('https://dog.ceo/api/breeds/list/all', async route => {
  const response = await route.fetch();
  const json = await response.json();
  json.message['big_red_dog'] = [];
  await route.fulfill({ response, json });
});
```

**Аргументы**
- `options` [Object] *(необязательный)*
  - `headers` [Object]&lt;[string], [string]&gt; *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fetch-option-headers"/><a href="#route-fetch-option-headers" class="list-anchor">#</a>
    
    Если задано, изменяет HTTP-заголовки запроса. Значения заголовков будут преобразованы в строку.
  - `maxRedirects` [number] *(необязательный)* <font size="2">Добавлено в: v1.31</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fetch-option-max-redirects"/><a href="#route-fetch-option-max-redirects" class="list-anchor">#</a>
    
    Максимальное количество перенаправлений запроса, которые будут автоматически выполнены. Если число превышено, будет выброшена ошибка. По умолчанию `20`. Установите `0`, чтобы не следовать перенаправлениям.
  - `maxRetries` [number] *(необязательный)* <font size="2">Добавлено в: v1.46</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fetch-option-max-retries"/><a href="#route-fetch-option-max-retries" class="list-anchor">#</a>
    
    Максимальное количество раз, которое следует повторить сетевые ошибки. В настоящее время повторяется только ошибка `ECONNRESET`. Не повторяет на основе кодов HTTP-ответов. Если лимит превышен, будет выброшена ошибка. По умолчанию `0` - без повторов.
  - `method` [string] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fetch-option-method"/><a href="#route-fetch-option-method" class="list-anchor">#</a>
    
    Если задано, изменяет метод запроса (например, GET или POST).
  - `postData` [string] | [Buffer] | [Serializable] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fetch-option-post-data"/><a href="#route-fetch-option-post-data" class="list-anchor">#</a>
    
    Позволяет установить данные поста запроса. Если параметр данных является объектом, он будет сериализован в строку JSON, и заголовок `content-type` будет установлен в `application/json`, если не установлен явно. В противном случае заголовок `content-type` будет установлен в `application/octet-stream`, если не установлен явно.
  - `timeout` [number] *(необязательный)* <font size="2">Добавлено в: v1.33</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fetch-option-timeout"/><a href="#route-fetch-option-timeout" class="list-anchor">#</a>
    
    Таймаут запроса в миллисекундах. По умолчанию `30000` (30 секунд). Установите `0`, чтобы отключить таймаут.
  - `url` [string] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fetch-option-url"/><a href="#route-fetch-option-url" class="list-anchor">#</a>
    
    Если задано, изменяет URL запроса. Новый URL должен иметь тот же протокол, что и оригинальный.

**Возвращает**
- [Promise]&lt;[APIResponse]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fetch-return"/><a href="#route-fetch-return" class="list-anchor">#</a>

**Подробности**

Обратите внимание, что опция [headers](/api/class-route.mdx#route-fetch-option-headers) будет применяться как к запрашиваемому запросу, так и к любым перенаправлениям, инициированным им. Если вы хотите применить [headers](/api/class-route.mdx#route-fetch-option-headers) только к оригинальному запросу, но не к перенаправлениям, посмотрите на [route.continue()](/api/class-route.mdx#route-continue).

### fulfill {#route-fulfill}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено до v1.9</font><x-search>route.fulfill</x-search>

Выполняет запрос маршрута с заданным ответом.

**Использование**

Пример выполнения всех запросов с ответами 404:

```js
await page.route('**/*', async route => {
  await route.fulfill({
    status: 404,
    contentType: 'text/plain',
    body: 'Не найдено!'
  });
});
```

Пример обслуживания статического файла:

```js
await page.route('**/xhr_endpoint', route => route.fulfill({ path: 'mock_data.json' }));
```

**Аргументы**
- `options` [Object] *(необязательный)*
  - `body` [string] | [Buffer] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fulfill-option-body"/><a href="#route-fulfill-option-body" class="list-anchor">#</a>
    
    Тело ответа.
  - `contentType` [string] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fulfill-option-content-type"/><a href="#route-fulfill-option-content-type" class="list-anchor">#</a>
    
    Если задано, эквивалентно установке заголовка ответа `Content-Type`.
  - `headers` [Object]&lt;[string], [string]&gt; *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fulfill-option-headers"/><a href="#route-fulfill-option-headers" class="list-anchor">#</a>
    
    Заголовки ответа. Значения заголовков будут преобразованы в строку.
  - `json` [Serializable] *(необязательный)* <font size="2">Добавлено в: v1.29</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fulfill-option-json"/><a href="#route-fulfill-option-json" class="list-anchor">#</a>
    
    JSON-ответ. Этот метод установит тип контента в `application/json`, если не установлен.
  - `path` [string] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fulfill-option-path"/><a href="#route-fulfill-option-path" class="list-anchor">#</a>
    
    Путь к файлу для ответа. Тип контента будет определен по расширению файла. Если `path` является относительным путем, он будет разрешен относительно текущего рабочего каталога.
  - `response` [APIResponse] *(необязательный)* <font size="2">Добавлено в: v1.15</font><a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fulfill-option-response"/><a href="#route-fulfill-option-response" class="list-anchor">#</a>
    
    [APIResponse], с которым нужно выполнить запрос маршрута. Отдельные поля ответа (такие как заголовки) могут быть переопределены с помощью параметров выполнения.
  - `status` [number] *(необязательный)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fulfill-option-status"/><a href="#route-fulfill-option-status" class="list-anchor">#</a>
    
    Код состояния ответа, по умолчанию `200`.

**Возвращает**
- [Promise]&lt;[void]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="route-fulfill-return"/><a href="#route-fulfill-return" class="list-anchor">#</a>
