Класс Touchscreen работает в координатах CSS пикселей главного фрейма относительно верхнего левого угла области просмотра. Методы сенсорного экрана могут использоваться только в контекстах браузера, которые были инициализированы с установленным значением `hasTouch` равным true.

Этот класс ограничен эмуляцией жестов касания. Для примеров других жестов, имитируемых вручную с помощью отправки событий касания, смотрите страницу [эмуляция устаревших событий касания](../touch-events.mdx).

---

## Методы

### tap {#touchscreen-tap}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено до v1.9</font><x-search>touchscreen.tap</x-search>

Отправляет события `touchstart` и `touchend` с одним касанием в позиции ([x](/api/class-touchscreen.mdx#touchscreen-tap-option-x),[y](/api/class-touchscreen.mdx#touchscreen-tap-option-y)).

:::note

Метод [page.tap()](/api/class-page.mdx#page-tap) вызовет ошибку, если опция [hasTouch](/api/class-browser.mdx#browser-new-context-option-has-touch) контекста браузера равна false.
:::

**Использование**

```js
await touchscreen.tap(x, y);
```

**Аргументы**
- `x` [number]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="touchscreen-tap-option-x"/><a href="#touchscreen-tap-option-x" class="list-anchor">#</a>
  
  Координата X относительно области просмотра главного фрейма в CSS пикселях.
- `y` [number]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="touchscreen-tap-option-y"/><a href="#touchscreen-tap-option-y" class="list-anchor">#</a>
  
  Координата Y относительно области просмотра главного фрейма в CSS пикселях.
