`Suite` — это группа тестов. Все тесты в Playwright Test формируют следующую иерархию:
* Корневая группа имеет дочернюю группу для каждого [FullProject].
  * Группа проекта #1. Имеет дочернюю группу для каждого файла тестов в проекте.
    * Группа файла #1
      * [TestCase] #1
      * [TestCase] #2
      * Группа, соответствующая [test.describe()](/api/class-test.mdx#test-describe)
        * [TestCase] #1 в группе
        * [TestCase] #2 в группе
      * < больше тестов ... >
    * Группа файла #2
    * < больше групп файлов ... >
  * Группа проекта #2
  * < больше групп проектов ... >

Репортер получает корневую группу в методе [reporter.onBegin()](/api/class-reporter.mdx#reporter-on-begin).

## Методы

### allTests {#suite-all-tests}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.10</font><x-search>suite.allTests</x-search>

Возвращает список всех тестовых случаев в этой группе и ее потомках, в отличие от [suite.tests](/api/class-suite.mdx#suite-tests).

**Использование**

```js
suite.allTests();
```

**Возвращает**
- [Array]&lt;[TestCase]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="suite-all-tests-return"/><a href="#suite-all-tests-return" class="list-anchor">#</a>

---

### entries {#suite-entries}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.44</font><x-search>suite.entries</x-search>

Тестовые случаи и группы, определенные непосредственно в этой группе. Элементы возвращаются в порядке их объявления. Вы можете различать различные типы записей, используя [testCase.type](/api/class-testcase.mdx#test-case-type) и [suite.type](/api/class-suite.mdx#suite-type).

**Использование**

```js
suite.entries();
```

**Возвращает**
- [Array]&lt;[TestCase] | [Suite]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="suite-entries-return"/><a href="#suite-entries-return" class="list-anchor">#</a>

---

### project {#suite-project}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.10</font><x-search>suite.project</x-search>

Конфигурация проекта, к которому принадлежит эта группа, или [void] для корневой группы.

**Использование**

```js
suite.project();
```

**Возвращает**
- [FullProject] | [undefined]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="suite-project-return"/><a href="#suite-project-return" class="list-anchor">#</a>

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.10</font><x-search>suite.titlePath</x-search>

Возвращает список названий от корня до этой группы.

**Использование**

```js
suite.titlePath();
```

**Возвращает**
- [Array]&lt;[string]&gt;<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="suite-title-path-return"/><a href="#suite-title-path-return" class="list-anchor">#</a>

## Свойства

### location {#suite-location}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.10</font><x-search>suite.location</x-search>

Местоположение в исходном коде, где определена группа. Отсутствует для корневых и проектных групп.

**Использование**

```js
suite.location
```

**Тип**
- [Location]

---

### parent {#suite-parent}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.10</font><x-search>suite.parent</x-search>

Родительская группа, отсутствует для корневой группы.

**Использование**

```js
suite.parent
```

**Тип**
- [Suite]

---

### suites {#suite-suites}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.10</font><x-search>suite.suites</x-search>

Дочерние группы. См. [Suite] для иерархии групп.

**Использование**

```js
suite.suites
```

**Тип**
- [Array]&lt;[Suite]&gt;

---

### tests {#suite-tests}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.10</font><x-search>suite.tests</x-search>

Тестовые случаи в группе. Обратите внимание, что в списке находятся только тестовые случаи, определенные непосредственно в этой группе. Любые тестовые случаи, определенные в вложенных группах [test.describe()](/api/class-test.mdx#test-describe), перечислены в дочерних [suite.suites](/api/class-suite.mdx#suite-suites).

**Использование**

```js
suite.tests
```

**Тип**
- [Array]&lt;[TestCase]&gt;

---

### title {#suite-title}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.10</font><x-search>suite.title</x-search>

Название группы.
* Пусто для корневой группы.
* Название проекта для проектной группы.
* Путь к файлу для группы файла.
* Название, переданное в [test.describe()](/api/class-test.mdx#test-describe) для групповой группы.

**Использование**

```js
suite.title
```

**Тип**
- [string]

---

### type {#suite-type}

<font size="2" style={{position: "relative", top: "-20px"}}>Добавлено в: v1.44</font><x-search>suite.type</x-search>

Возвращает тип группы. Группы формируют следующую иерархию: `root` -> `project` -> `file` -> `describe` -> ...`describe` -> `test`.

**Использование**

```js
suite.type
```

**Тип**
- "root" | "project" | "file" | "describe"
