## Введение

С помощью Playwright вы можете запускать один тест, набор тестов или все тесты. Тесты могут выполняться в одном браузере или нескольких браузерах с использованием флага `--project`. По умолчанию тесты запускаются параллельно и в безголовом режиме, что означает, что окно браузера не будет открыто во время выполнения тестов, а результаты будут отображаться в терминале. Однако вы можете запустить тесты в обычном режиме, используя аргумент CLI `--headed`, или вы можете запустить тесты в [UI режиме](./test-ui-mode.mdx), используя флаг `--ui`. Посмотрите полный трассировочный отчет о ваших тестах с возможностью просмотра в реальном времени, отладки с перемоткой времени и многим другим.

**Вы узнаете**
- [Как запускать тесты из командной строки](/running-tests.mdx#command-line)
- [Как отлаживать тесты](/running-tests.mdx#debugging-tests)
- [Как открыть HTML-репортер тестов](/running-tests.mdx#test-reports)

## Запуск тестов

### Командная строка

Вы можете запустить свои тесты с помощью команды `playwright test`. Это запустит ваши тесты во всех браузерах, как указано в файле `playwright.config`. Тесты по умолчанию запускаются в безголовом режиме, что означает, что окно браузера не будет открыто во время выполнения тестов, а результаты будут отображаться в терминале.

```bash
npx playwright test
```

![тесты, выполняющиеся в командной строке](https://github.com/microsoft/playwright/assets/13063165/981c1b2b-dc7e-4b85-b241-272b44da6628)

### Запуск тестов в UI режиме

Мы настоятельно рекомендуем запускать ваши тесты в [UI режиме](./test-ui-mode.mdx) для лучшего опыта разработчика, где вы можете легко пройти через каждый шаг теста и визуально увидеть, что происходило до, во время и после каждого шага. UI режим также предлагает множество других функций, таких как выбор локатора, режим просмотра и многое другое.

```bash
npx playwright test --ui
```

![UI режим](https://github.com/microsoft/playwright/assets/13063165/c5b501cc-4f5d-485a-87cc-66044c651786)

Посмотрите наш [подробный гид по UI режиму](./test-ui-mode.mdx), чтобы узнать больше о его функциях.

### Запуск тестов в обычном режиме

Чтобы запустить ваши тесты в обычном режиме, используйте флаг `--headed`. Это позволит вам визуально увидеть, как Playwright взаимодействует с веб-сайтом.

```bash
npx playwright test --headed
```

### Запуск тестов в разных браузерах

Чтобы указать, в каком браузере вы хотите запустить свои тесты, используйте флаг `--project`, за которым следует имя браузера.

```bash
npx playwright test --project webkit
```

Чтобы указать несколько браузеров для запуска ваших тестов, используйте флаг `--project` несколько раз, за которым следует имя каждого браузера.

```bash
npx playwright test --project webkit --project firefox
```

### Запуск конкретных тестов

Чтобы запустить один файл теста, передайте имя файла теста, который вы хотите запустить.

```bash
npx playwright test landing-page.spec.ts
```

Чтобы запустить набор файлов тестов из разных каталогов, передайте имена каталогов, в которых вы хотите запустить тесты.

```bash
npx playwright test tests/todo-page/ tests/landing-page/
```

Чтобы запустить файлы, в имени которых есть `landing` или `login`, просто передайте эти ключевые слова в CLI.

```bash
npx playwright test landing login
```

Чтобы запустить тест с конкретным заголовком, используйте флаг `-g`, за которым следует заголовок теста.

```bash
npx playwright test -g "добавить элемент в список дел"
```

### Запуск последних неудачных тестов

Чтобы запустить только тесты, которые не прошли в последнем запуске тестов, сначала запустите свои тесты, а затем запустите их снова с флагом `--last-failed`.

```bash
npx playwright test --last-failed
```

### Запуск тестов в VS Code

Тесты можно запускать прямо из VS Code с помощью [расширения VS Code](https://marketplace.visualstudio.com/items?itemName=ms-playwright.playwright). После установки вы можете просто нажать на зеленый треугольник рядом с тестом, который хотите запустить, или запустить все тесты из боковой панели тестирования. Ознакомьтесь с нашим [руководством по началу работы с VS Code](./getting-started-vscode.mdx#running-tests) для получения дополнительных деталей.

![Расширение Playwright для VS Code](https://github.com/microsoft/playwright/assets/13063165/47726e70-683b-4bd5-94de-7d03dd45c30f)

## Отладка тестов

Поскольку Playwright работает в Node.js, вы можете отлаживать его с помощью вашего любимого отладчика, например, используя `console.log` или внутри вашей IDE, или прямо в VS Code с помощью [расширения VS Code](./getting-started-vscode.mdx). Playwright поставляется с [UI режимом](./test-ui-mode.mdx), где вы можете легко пройти через каждый шаг теста, видеть логи, ошибки, сетевые запросы, инспектировать снимок DOM и многое другое. Вы также можете использовать [Playwright Inspector](./debug.mdx#playwright-inspector), который позволяет вам пошагово проходить вызовы API Playwright, видеть их отладочные логи и исследовать [локаторы](./locators.mdx).

### Отладка тестов в UI режиме

Мы настоятельно рекомендуем отлаживать ваши тесты в [UI режиме](./test-ui-mode.mdx) для лучшего опыта разработчика, где вы можете легко пройти через каждый шаг теста и визуально увидеть, что происходило до, во время и после каждого шага. UI режим также предлагает множество других функций, таких как выбор локатора, режим просмотра и многое другое.

```bash
npx playwright test --ui
```

![показ ошибок в ui режиме](https://github.com/microsoft/playwright/assets/13063165/ffca2fd1-5349-41fb-ade9-ace143bb2c58)

Во время отладки вы можете использовать кнопку "Выбрать локатор", чтобы выбрать элемент на странице и увидеть локатор, который Playwright использует для поиска этого элемента. Вы также можете редактировать локатор в игровом пространстве локаторов и видеть, как он выделяется в реальном времени в окне браузера. Используйте кнопку "Копировать локатор", чтобы скопировать локатор в буфер обмена, а затем вставьте его в ваш тест.

![выбор локатора в ui режиме](https://github.com/microsoft/playwright/assets/13063165/9e7eeb84-bd26-4010-8614-75e24b56c716)

Посмотрите наш [подробный гид по UI режиму](./test-ui-mode.mdx), чтобы узнать больше о его функциях.

### Отладка тестов с помощью Playwright Inspector

Чтобы отладить все тесты, выполните команду теста Playwright, добавив флаг `--debug`.

```bash
npx playwright test --debug
```

![Отладка тестов с помощью Playwright Inspector](https://github.com/microsoft/playwright/assets/13063165/6b3b3caa-d258-4cb8-aa05-cd407f501626)

Эта команда откроет окно браузера, а также Playwright Inspector. Вы можете использовать кнопку "Шаг через" в верхней части инспектора, чтобы пройти через ваш тест. Или нажмите кнопку "Воспроизвести", чтобы запустить тест от начала до конца. После завершения теста окно браузера закроется.

Чтобы отладить один файл теста, выполните команду теста Playwright с именем файла теста, который вы хотите отладить, добавив флаг `--debug`.

```bash
npx playwright test example.spec.ts --debug
```

Чтобы отладить конкретный тест с номера строки, где определен `test(..`, добавьте двоеточие, за которым следует номер строки в конце имени файла теста, затем добавьте флаг `--debug`.

```bash
npx playwright test example.spec.ts:10 --debug
```

Во время отладки вы можете использовать кнопку "Выбрать локатор", чтобы выбрать элемент на странице и увидеть локатор, который Playwright использует для поиска этого элемента. Вы также можете редактировать локатор и видеть, как он выделяется в реальном времени в окне браузера. Используйте кнопку "Копировать локатор", чтобы скопировать локатор в буфер обмена, а затем вставьте его в ваш тест.

![Выбор локатора в Playwright Inspector](https://github.com/microsoft/playwright/assets/13063165/013d5edb-583e-423a-bb53-9f57bab7f3e1)

Посмотрите наш [гид по отладке](./debug.mdx), чтобы узнать больше об отладке с помощью [отладчика VS Code](./debug.mdx#vs-code-debugger), UI режима и [Playwright Inspector](./debug.mdx#playwright-inspector), а также об отладке с помощью [инструментов разработчика браузера](./debug.mdx#browser-developer-tools).

## Отчеты о тестах

[HTML-репортер](./test-reporters.mdx#html-reporter) показывает полный отчет о ваших тестах, позволяя фильтровать отчет по браузерам, пройденным тестам, неудачным тестам, пропущенным тестам и ненадежным тестам. По умолчанию HTML-отчет открывается автоматически, если некоторые тесты не прошли, в противном случае вы можете открыть его с помощью следующей команды.

```bash
npx playwright show-report
```

![HTML отчет](https://github.com/microsoft/playwright/assets/13063165/c5f60e56-fb75-4a2d-a4b6-054b8c5d69c1)

Вы можете фильтровать и искать тесты, а также щелкать по каждому тесту, чтобы увидеть ошибки тестов и исследовать каждый шаг теста.

![Подробный вид HTML репортера](https://github.com/microsoft/playwright/assets/13063165/f36ada14-4701-4534-a3be-ed22c2b16cf5)
