Playwright Test имеет несколько настраиваемых таймаутов для различных задач.

| Таймаут          | По умолчанию      | Описание                          |
|:-----------------|:-----------------|:----------------------------------|
| Таймаут теста    | 30_000 мс        | Таймаут для каждого теста<br/><span style={{textTransform:'uppercase',fontSize:'smaller',fontWeight:'bold',opacity:'0.7'}}>Установить в конфигурации</span><br/><code>{`{ timeout: 60_000 }`}</code><br/><span style={{textTransform: 'uppercase',fontSize: 'smaller', fontWeight: 'bold', opacity: '0.7'}}>Переопределить в тесте</span><br/>`test.setTimeout(120_000)` |
| Таймаут ожидания | 5_000 мс         | Таймаут для каждого утверждения<br/><span style={{textTransform:'uppercase',fontSize:'smaller',fontWeight:'bold',opacity:'0.7'}}>Установить в конфигурации</span><br/><code>{`{ expect: { timeout: 10_000 } }`}</code><br/><span style={{textTransform: 'uppercase',fontSize: 'smaller', fontWeight: 'bold', opacity: '0.7'}}>Переопределить в тесте</span><br/>`expect(locator).toBeVisible({ timeout: 10_000 })` |

## Таймаут теста

Playwright Test устанавливает таймаут для каждого теста, по умолчанию 30 секунд. Время, затраченное на выполнение функции теста, настройку фикстур и хуки `beforeEach`, включается в таймаут теста.

Тест, который превысил таймаут, выдает следующую ошибку:

```txt
example.spec.ts:3:1 › базовый тест ===========================

Таймаут в 30000 мс превышен.
```

Дополнительный отдельный таймаут, равный этому значению, используется для завершения фикстур и хуков `afterEach`, после завершения функции теста.

То же значение таймаута также применяется к хукам `beforeAll` и `afterAll`, но они не делят время с каким-либо тестом.

### Установить таймаут теста в конфигурации

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  timeout: 120_000,
});
```

Справочная информация по API: [testConfig.timeout](/api/class-testconfig.mdx#test-config-timeout).

### Установить таймаут для одного теста

```js title="example.spec.ts"
import { test, expect } from '@playwright/test';

test('медленный тест', async ({ page }) => {
  test.slow(); // Легкий способ утроить таймаут по умолчанию
  // ...
});

test('очень медленный тест', async ({ page }) => {
  test.setTimeout(120_000);
  // ...
});
```

Справочная информация по API: [test.setTimeout()](/api/class-test.mdx#test-set-timeout) и [test.slow()](/api/class-test.mdx#test-slow).

### Изменить таймаут из хука `beforeEach`

```js title="example.spec.ts"
import { test, expect } from '@playwright/test';

test.beforeEach(async ({ page }, testInfo) => {
  // Увеличить таймаут для всех тестов, использующих этот хук, на 30 секунд.
  testInfo.setTimeout(testInfo.timeout + 30_000);
});
```

Справочная информация по API: [testInfo.setTimeout()](/api/class-testinfo.mdx#test-info-set-timeout).

### Изменить таймаут для хуков `beforeAll`/`afterAll`

Хуки `beforeAll` и `afterAll` имеют отдельный таймаут, по умолчанию равный таймауту теста. Вы можете изменить его отдельно для каждого хука, вызвав [testInfo.setTimeout()](/api/class-testinfo.mdx#test-info-set-timeout) внутри хука.

```js title="example.spec.ts"
import { test, expect } from '@playwright/test';

test.beforeAll(async () => {
  // Установить таймаут для этого хука.
  test.setTimeout(60000);
});
```

Справочная информация по API: [testInfo.setTimeout()](/api/class-testinfo.mdx#test-info-set-timeout).

## Таймаут ожидания

Авто-повторяющиеся утверждения, такие как [expect(locator).toHaveText()](/api/class-locatorassertions.mdx#locator-assertions-to-have-text), имеют отдельный таймаут, по умолчанию 5 секунд. Таймаут утверждения не связан с таймаутом теста. Он выдает следующую ошибку:

```txt
example.spec.ts:3:1 › базовый тест ===========================

Ошибка: expect(received).toHaveText(expected)

Ожидаемая строка: "my text"
Полученная строка: ""
Журнал вызовов:
  - expect.toHaveText с таймаутом 5000 мс
  - ожидание "locator('button')"
```

### Установить таймаут ожидания в конфигурации

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  expect: {
    timeout: 10_000,
  },
});
```

Справочная информация по API: [testConfig.expect](/api/class-testconfig.mdx#test-config-expect).

### Указать таймаут ожидания для одного утверждения

```js title="example.spec.ts"
import { test, expect } from '@playwright/test';

test('пример', async ({ page }) => {
  await expect(locator).toHaveText('hello', { timeout: 10_000 });
});
```

## Глобальный таймаут

Playwright Test поддерживает таймаут для всего запуска тестов. Это предотвращает избыточное использование ресурсов, когда что-то пошло не так. Глобальный таймаут по умолчанию не установлен, но вы можете установить разумный таймаут в конфигурации, например, на один час. Глобальный таймаут выдает следующую ошибку:

```txt
Запуск 1000 тестов с использованием 10 рабочих процессов

  514 пропущено
  486 пройдено
  Таймаут ожидания 3600 с для всего запуска тестов
```

Вы можете установить глобальный таймаут в конфигурации.

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  globalTimeout: 3_600_000,
});
```

Справочная информация по API: [testConfig.globalTimeout](/api/class-testconfig.mdx#test-config-global-timeout).

## Расширенные: низкоуровневые таймауты

Это низкоуровневые таймауты, которые предварительно настроены тестовым раннером, и вам не нужно их изменять. Если вы находитесь в этом разделе из-за нестабильности ваших тестов, скорее всего, вам следует искать решение в другом месте.

| Таймаут                     | По умолчанию      | Описание                          |
|:----------------------------|:-----------------|:----------------------------------|
| Таймаут действия            | нет таймаута     | Таймаут для каждого действия<br/><span style={{textTransform:'uppercase',fontSize:'smaller',fontWeight:'bold',opacity:'0.7'}}>Установить в конфигурации</span><br/><code>{`{ use: { actionTimeout: 10_000 } }`}</code><br/><span style={{textTransform: 'uppercase',fontSize: 'smaller',fontWeight: 'bold',opacity:'0.7'}}>Переопределить в тесте</span><br/>`locator.click({ timeout: 10_000 })` |
| Таймаут навигации          | нет таймаута     | Таймаут для каждого действия навигации<br/><span style={{textTransform:'uppercase',fontSize:'smaller',fontWeight:'bold',opacity:'0.7'}}>Установить в конфигурации</span><br/><code>{`{ use: { navigationTimeout: 30_000 } }`}</code><br/><span style={{textTransform: 'uppercase',fontSize: 'smaller',fontWeight: 'bold',opacity:'0.7'}}>Переопределить в тесте</span><br/>`page.goto('/', { timeout: 30_000 })` |
| Глобальный таймаут         | нет таймаута     | Глобальный таймаут для всего запуска тестов<br/><span style={{textTransform:'uppercase',fontSize:'smaller',fontWeight:'bold',opacity:'0.7'}}>Установить в конфигурации</span><br/>`{ globalTimeout: 3_600_000 }`<br/> |
| Таймаут `beforeAll`/`afterAll` | 30_000 мс      | Таймаут для хука<br/><span style={{textTransform:'uppercase',fontSize:'smaller',fontWeight:'bold',opacity:'0.7'}}>Установить в хуке</span><br/>`test.setTimeout(60_000)`<br/> |
| Таймаут фикстуры           | нет таймаута     | Таймаут для отдельной фикстуры<br/><span style={{textTransform:'uppercase',fontSize:'smaller',fontWeight:'bold',opacity:'0.7'}}>Установить в фикстуре</span><br/>`{ scope: 'test', timeout: 30_000 }`<br/> |

### Установить таймауты действий и навигации в конфигурации

```js title="playwright.config.ts"
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    actionTimeout: 10 * 1000,
    navigationTimeout: 30 * 1000,
  },
});
```

Справочная информация по API: [testOptions.actionTimeout](/api/class-testoptions.mdx#test-options-action-timeout) и [testOptions.navigationTimeout](/api/class-testoptions.mdx#test-options-navigation-timeout).

### Установить таймаут для одного действия

```js title="example.spec.ts"
import { test, expect } from '@playwright/test';

test('базовый тест', async ({ page }) => {
  await page.goto('https://playwright.dev', { timeout: 30000 });
  await page.getByText('Начать работу').click({ timeout: 10000 });
});
```

## Таймаут фикстуры

По умолчанию [фикстура](./test-fixtures) делит таймаут с тестом. Однако для медленных фикстур, особенно [фикстур с областью действия рабочего процесса](./test-fixtures#worker-scoped-fixtures), удобно иметь отдельный таймаут. Таким образом, вы можете сохранить общий таймаут теста небольшим и дать медленной фикстуре больше времени.

```js title="example.spec.ts"
import { test as base, expect } from '@playwright/test';

const test = base.extend<{ slowFixture: string }>({
  slowFixture: [async ({}, use) => {
    // ... выполнить медленную операцию ...
    await use('hello');
  }, { timeout: 60_000 }]
});

test('пример теста', async ({ slowFixture }) => {
  // ...
});
```

Справочная информация по API: [test.extend()](/api/class-test.mdx#test-extend).
