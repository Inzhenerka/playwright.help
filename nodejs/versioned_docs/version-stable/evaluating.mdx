## Введение

Скрипты Playwright выполняются в вашей среде Playwright. Скрипты вашей страницы выполняются в среде браузера. Эти среды не пересекаются, они работают в разных виртуальных машинах в разных процессах и даже потенциально на разных компьютерах.

API [page.evaluate()](/api/class-page.mdx#page-evaluate) может выполнять функцию JavaScript в контексте веб-страницы и возвращать результаты обратно в среду Playwright. Глобальные объекты браузера, такие как `window` и `document`, могут использоваться в `evaluate`.

```js
const href = await page.evaluate(() => document.location.href);
```

Если результат является Promise или если функция асинхронная, evaluate автоматически будет ждать, пока он не будет разрешен:

```js
const status = await page.evaluate(async () => {
  const response = await fetch(location.href);
  return response.status;
});
```

## Разные среды

Оцененные скрипты выполняются в среде браузера, в то время как ваш тест выполняется в тестовой среде. Это означает, что вы не можете использовать переменные из вашего теста на странице и наоборот. Вместо этого вы должны передавать их явно в качестве аргумента.

Следующий фрагмент кода **НЕПРАВИЛЬНЫЙ**, потому что он использует переменную напрямую:

```js
const data = 'some data';
const result = await page.evaluate(() => {
  // НЕПРАВИЛЬНО: "data" нет на веб-странице.
  window.myApp.use(data);
});
```

Следующий фрагмент кода **ПРАВИЛЬНЫЙ**, потому что он передает значение явно в качестве аргумента:

```js
const data = 'some data';
// Передаем |data| как параметр.
const result = await page.evaluate(data => {
  window.myApp.use(data);
}, data);
```

## Аргумент оценки

Методы оценки Playwright, такие как [page.evaluate()](/api/class-page.mdx#page-evaluate), принимают один необязательный аргумент. Этот аргумент может быть смесью значений [Serializable] и экземпляров [JSHandle]. Хэндлы автоматически преобразуются в значение, которое они представляют.

```js
// Примитивное значение.
await page.evaluate(num => num, 42);

// Массив.
await page.evaluate(array => array.length, [1, 2, 3]);

// Объект.
await page.evaluate(object => object.foo, { foo: 'bar' });

// Один хэндл.
const button = await page.evaluateHandle('window.button');
await page.evaluate(button => button.textContent, button);

// Альтернативная нотация с использованием JSHandle.evaluate.
await button.evaluate((button, from) => button.textContent.substring(from), 5);

// Объект с несколькими хэндлами.
const button1 = await page.evaluateHandle('window.button1');
const button2 = await page.evaluateHandle('window.button2');
await page.evaluate(
    o => o.button1.textContent + o.button2.textContent,
    { button1, button2 });

// Деструктуризация объекта работает. Обратите внимание, что имена свойств должны совпадать
// между деструктурированным объектом и аргументом.
// Также обратите внимание на обязательные скобки.
await page.evaluate(
    ({ button1, button2 }) => button1.textContent + button2.textContent,
    { button1, button2 });

// Массив также работает. Для деструктуризации можно использовать произвольные имена.
// Обратите внимание на обязательные скобки.
await page.evaluate(
    ([b1, b2]) => b1.textContent + b2.textContent,
    [button1, button2]);

// Любая смесь сериализуемых значений и хэндлов работает.
await page.evaluate(
    x => x.button1.textContent + x.list[0].textContent + String(x.foo),
    { button1, list: [button2], foo: null });
```

## Инициализационные скрипты

Иногда удобно оценивать что-то на странице до того, как она начнет загружаться. Например, вы можете захотеть настроить некоторые моки или тестовые данные.

В этом случае используйте [page.addInitScript()](/api/class-page.mdx#page-add-init-script) или [browserContext.addInitScript()](/api/class-browsercontext.mdx#browser-context-add-init-script). В приведенном ниже примере мы заменим `Math.random()` на постоянное значение.

Сначала создайте файл `preload.js`, который содержит мок.

```js
// preload.js
Math.random = () => 42;
```

Затем добавьте инициализационный скрипт на страницу.

```js
import { test, expect } from '@playwright/test';
import path from 'path';

test.beforeEach(async ({ page }) => {
  // Добавьте скрипт для каждого теста в хуке beforeEach.
  // Убедитесь, что путь к скрипту правильно разрешен.
  await page.addInitScript({ path: path.resolve(__dirname, '../mocks/preload.js') });
});
```

###### 
В качестве альтернативы вы можете передать функцию вместо создания файла скрипта предварительной загрузки. Это более удобно для коротких или одноразовых скриптов. Вы также можете передать аргумент таким образом.

```js
import { test, expect } from '@playwright/test';

// Добавьте скрипт для каждого теста в хуке beforeEach.
test.beforeEach(async ({ page }) => {
  const value = 42;
  await page.addInitScript(value => {
    Math.random = () => value;
  }, value);
});
```
