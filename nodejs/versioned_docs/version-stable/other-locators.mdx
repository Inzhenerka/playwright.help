## Введение

:::note

Посмотрите основной [гид по локаторам](./locators) для наиболее распространенных и рекомендуемых локаторов.
:::

В дополнение к рекомендуемым локаторам, таким как [page.getByRole()](/api/class-page.mdx#page-get-by-role) и [page.getByText()](/api/class-page.mdx#page-get-by-text), Playwright поддерживает различные другие локаторы, описанные в этом руководстве.

## CSS локатор

:::note

Мы рекомендуем отдавать предпочтение [видимым пользователю локаторам](./locators.mdx#quick-guide), таким как текст или доступная роль, вместо использования CSS, который связан с реализацией и может сломаться при изменении страницы.
:::

Playwright может находить элемент по CSS-селектору.

```js
await page.locator('css=button').click();
```

Playwright расширяет стандартные CSS-селекторы двумя способами:
* CSS-селекторы проникают в открытый shadow DOM.
* Playwright добавляет пользовательские псевдоклассы, такие как `:visible`, `:has-text()`, `:has()`, `:is()`, `:nth-match()` и другие.

### CSS: соответствие по тексту

Playwright включает ряд CSS-псевдоклассов для соответствия элементов по их текстовому содержимому.
- `article:has-text("Playwright")` - `:has-text()` соответствует любому элементу, содержащему указанный текст где-то внутри, возможно, в дочернем или потомственном элементе. Сравнение нечувствительно к регистру, обрезает пробелы и ищет подстроку.
  
  Например, `article:has-text("Playwright")` соответствует `<article><div>Playwright</div></article>`.
  
  Обратите внимание, что `:has-text()` следует использовать вместе с другими CSS-спецификаторами, иначе он будет соответствовать всем элементам, содержащим указанный текст, включая `<body>`.
  
  ```js
  // Неправильно, будет соответствовать многим элементам, включая <body>
  await page.locator(':has-text("Playwright")').click();
  // Правильно, соответствует только элементу <article>
  await page.locator('article:has-text("Playwright")').click();
  ```
  
- `#nav-bar :text("Home")` - псевдокласс `:text()` соответствует наименьшему элементу, содержащему указанный текст. Сравнение нечувствительно к регистру, обрезает пробелы и ищет подстроку.
  
  Например, это найдет элемент с текстом "Home" где-то внутри элемента `#nav-bar`:
  
  ```js
  await page.locator('#nav-bar :text("Home")').click();
  ```
  
- `#nav-bar :text-is("Home")` - псевдокласс `:text-is()` соответствует наименьшему элементу с точным текстом. Точное соответствие чувствительно к регистру, обрезает пробелы и ищет полную строку.
  
  Например, `:text-is("Log")` не соответствует `<button>Log in</button>`, потому что `<button>` содержит один текстовый узел `"Log in"`, который не равен `"Log"`. Однако `:text-is("Log")` соответствует `<button> Log <span>in</span></button>`, потому что `<button>` содержит текстовый узел `" Log "`.
  
  Аналогично, `:text-is("Download")` не будет соответствовать `<button>download</button>`, потому что это чувствительно к регистру.
* `#nav-bar :text-matches("reg?ex", "i")` - псевдокласс `:text-matches()` соответствует наименьшему элементу с текстовым содержимым, соответствующим [регулярному выражению, похожему на JavaScript](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/RegExp).
  
  Например, `:text-matches("Log\s*in", "i")` соответствует `<button>Login</button>` и `<button>log IN</button>`.

:::note

Сравнение текста всегда нормализует пробелы. Например, оно превращает несколько пробелов в один, заменяет переносы строк на пробелы и игнорирует начальные и конечные пробелы.
:::

:::note
Элементы ввода типа `button` и `submit` соответствуют по своему `value`, а не по текстовому содержимому. Например, `:text("Log in")` соответствует `<input type=button value="Log in">`.
:::

### CSS: соответствие только видимым элементам

Playwright поддерживает псевдокласс `:visible` в CSS-селекторах. Например, `css=button` соответствует всем кнопкам на странице, в то время как `css=button:visible` соответствует только видимым кнопкам. Это полезно для различения элементов, которые очень похожи, но отличаются по видимости.

Рассмотрим страницу с двумя кнопками, первая невидима, а вторая видима.

```html
<button style='display: none'>Невидимая</button>
<button>Видимая</button>
```

* Это найдет обе кнопки и вызовет ошибку [строгости](./locators.mdx#strictness):
  
  ```js
  await page.locator('button').click();
  ```
  
* Это найдет только вторую кнопку, потому что она видима, и затем кликнет по ней.
  
  ```js
  await page.locator('button:visible').click();
  ```
  
### CSS: элементы, содержащие другие элементы

Псевдокласс `:has()` является [экспериментальным CSS-псевдоклассом](https://developer.mozilla.org/en-US/docs/Web/CSS/:has). Он возвращает элемент, если любой из селекторов, переданных в качестве параметров относительно `:scope` данного элемента, соответствует хотя бы одному элементу.

Следующий фрагмент возвращает текстовое содержимое элемента `<article>`, который содержит `<div class=promo>` внутри.

```js
await page.locator('article:has(div.promo)').textContent();
```

### CSS: элементы, соответствующие одному из условий

Список CSS-селекторов, разделенный запятыми, будет соответствовать всем элементам, которые могут быть выбраны одним из селекторов в этом списке.

```js
// Кликает по <button>, который имеет текст "Log in" или "Sign in".
await page.locator('button:has-text("Log in"), button:has-text("Sign in")').click();
```

Псевдокласс `:is()` является [экспериментальным CSS-псевдоклассом](https://developer.mozilla.org/en-US/docs/Web/CSS/:is), который может быть полезен для указания списка дополнительных условий для элемента.

### CSS: соответствие элементов на основе расположения

:::note

Соответствие на основе расположения может привести к неожиданным результатам. Например, другой элемент может быть соответствующим, когда расположение изменяется на один пиксель.
:::

Иногда трудно придумать хороший селектор для целевого элемента, когда у него нет отличительных признаков. В этом случае использование CSS-псевдоклассов расположения Playwright может помочь. Их можно комбинировать с обычным CSS, чтобы точно указать один из нескольких вариантов.

Например, `input:right-of(:text("Password"))` соответствует полю ввода, которое находится справа от текста "Password" - полезно, когда на странице есть несколько полей ввода, которые трудно различить.

Обратите внимание, что псевдоклассы расположения полезны в дополнение к чему-то еще, например, `input`. Если вы используете псевдокласс расположения отдельно, например, `:right-of(:text("Password"))`, скорее всего, вы получите не то поле ввода, которое ищете, а какой-то пустой элемент между текстом и целевым полем ввода.

Псевдоклассы расположения используют [bounding client rect](https://developer.mozilla.org/en-US/docs/Web/API/Element/getBoundingClientRect) для вычисления расстояния и относительного положения элементов.
* `:right-of(div > button)` - соответствует элементам, которые находятся справа от любого элемента, соответствующего внутреннему селектору, на любом вертикальном уровне.
* `:left-of(div > button)` - соответствует элементам, которые находятся слева от любого элемента, соответствующего внутреннему селектору, на любом вертикальном уровне.
* `:above(div > button)` - соответствует элементам, которые находятся выше любого из элементов, соответствующих внутреннему селектору, на любом горизонтальном уровне.
* `:below(div > button)` - соответствует элементам, которые находятся ниже любого из элементов, соответствующих внутреннему селектору, на любом горизонтальном уровне.
* `:near(div > button)` - соответствует элементам, которые находятся рядом (в пределах 50 CSS пикселей) с любым из элементов, соответствующих внутреннему селектору.

Обратите внимание, что полученные соответствия сортируются по расстоянию до опорного элемента, поэтому вы можете использовать [locator.first()](/api/class-locator.mdx#locator-first), чтобы выбрать ближайший. Это полезно только в случае, если у вас есть что-то вроде списка похожих элементов, где ближайший очевидно является правильным. Однако использование [locator.first()](/api/class-locator.mdx#locator-first) в других случаях, скорее всего, не сработает так, как ожидалось - он не будет нацеливаться на элемент, который вы ищете, а на какой-то другой элемент, который оказывается ближайшим, например, на случайный пустой `<div>`, или на элемент, который прокручен и в данный момент не виден.

```js
// Заполнить поле ввода справа от "Username".
await page.locator('input:right-of(:text("Username"))').fill('value');

// Кликнуть по кнопке рядом с промо-картой.
await page.locator('button:near(.promo-card)').click();

// Кликнуть по радио-кнопке в списке, ближайшей к "Label 3".
await page.locator('[type=radio]:left-of(:text("Label 3"))').first().click();
```

Все псевдоклассы расположения поддерживают необязательное максимальное расстояние в пикселях в качестве последнего аргумента. Например, `button:near(:text("Username"), 120)` соответствует кнопке, которая находится на расстоянии не более 120 CSS пикселей от элемента с текстом "Username".

### CSS: выбор n-го совпадения из результата запроса

:::note

Обычно возможно различить элементы по какому-либо атрибуту или текстовому содержимому, что более устойчиво к изменениям страницы.
:::

Иногда на странице содержится несколько похожих элементов, и трудно выбрать конкретный. Например:

```html
<section> <button>Купить</button> </section>
<article><div> <button>Купить</button> </div></article>
<div><div> <button>Купить</button> </div></div>
```

В этом случае `:nth-match(:text("Купить"), 3)` выберет третью кнопку из приведенного выше фрагмента. Обратите внимание, что индекс начинается с единицы.

```js
// Кликнуть по третьей кнопке "Купить"
await page.locator(':nth-match(:text("Купить"), 3)').click();
```

`:nth-match()` также полезен для ожидания, пока указанное количество элементов появится, используя [locator.waitFor()](/api/class-locator.mdx#locator-wait-for).

```js
// Ожидать, пока все три кнопки станут видимыми
await page.locator(':nth-match(:text("Купить"), 3)').waitFor();
```

:::note
В отличие от [`:nth-child()`](https://developer.mozilla.org/en-US/docs/Web/CSS/:nth-child), элементы не обязательно должны быть соседями, они могут находиться где угодно на странице. В приведенном выше фрагменте все три кнопки соответствуют селектору `:text("Купить")`, и `:nth-match()` выбирает третью кнопку.
:::

## Локатор n-го элемента

Вы можете сузить запрос до n-го совпадения, используя локатор `nth=`, передавая индекс, начинающийся с нуля.

```js
// Кликнуть по первой кнопке
await page.locator('button').locator('nth=0').click();

// Кликнуть по последней кнопке
await page.locator('button').locator('nth=-1').click();
```

## Локатор родительского элемента

Когда вам нужно нацелиться на родительский элемент какого-либо другого элемента, чаще всего следует использовать [locator.filter()](/api/class-locator.mdx#locator-filter) по локатору дочернего элемента. Например, рассмотрим следующую структуру DOM:

```html
<li><label>Привет</label></li>
<li><label>Мир</label></li>
```

Если вы хотите нацелиться на родительский `<li>` метки с текстом `"Привет"`, использование [locator.filter()](/api/class-locator.mdx#locator-filter) будет наиболее эффективным:

```js
const child = page.getByText('Привет');
const parent = page.getByRole('listitem').filter({ has: child });
```

В качестве альтернативы, если вы не можете найти подходящий локатор для родительского элемента, используйте `xpath=..`. Обратите внимание, что этот метод не так надежен, потому что любые изменения в структуре DOM сломают ваши тесты. Предпочитайте [locator.filter()](/api/class-locator.mdx#locator-filter), когда это возможно.

```js
const parent = page.getByText('Привет').locator('xpath=..');
```

## Локатор React

:::note

Локатор React является экспериментальным и имеет префикс `_`. Функциональность может измениться в будущем.
:::

Локатор React позволяет находить элементы по имени компонента и значениям свойств. Синтаксис очень похож на [CSS-селекторы атрибутов](https://developer.mozilla.org/en-US/docs/Web/CSS/Attribute_selectors) и поддерживает все операторы селекторов атрибутов CSS.

В локаторе React имена компонентов записываются в **CamelCase**.

```js
await page.locator('_react=BookItem').click();
```

Больше примеров:
- соответствие по **компоненту**: `_react=BookItem`
- соответствие по компоненту и **точному значению свойства**, чувствительному к регистру: `_react=BookItem[author = "Steven King"]`
- соответствие только по значению свойства, **недоступному к регистру**: `_react=[author = "steven king" i]`
- соответствие по компоненту и **истинному значению свойства**: `_react=MyButton[enabled]`
- соответствие по компоненту и **логическому значению**: `_react=MyButton[enabled = false]`
- соответствие по значению свойства **подстроки**: `_react=[author *= "King"]`
- соответствие по компоненту и **нескольким свойствам**: `_react=BookItem[author *= "king" i][year = 1990]`
- соответствие по **вложенному** значению свойства: `_react=[some.nested.value = 12]`
- соответствие по компоненту и значению свойства **префикса**: `_react=BookItem[author ^= "Steven"]`
- соответствие по компоненту и значению свойства **суффикса**: `_react=BookItem[author $= "Steven"]`
- соответствие по компоненту и **ключу**: `_react=BookItem[key = '2']`
- соответствие по значению свойства **regex**: `_react=[author = /Steven(\\s+King)?/i]`

Чтобы найти имена элементов React в дереве, используйте [React DevTools](https://chrome.google.com/webstore/detail/react-developer-tools/fmkadmapgofadopljbjfkapdkoienihi).

:::note

Локатор React поддерживает React 15 и выше.
:::

:::note
Локатор React, а также [React DevTools](https://chrome.google.com/webstore/detail/react-developer-tools/fmkadmapgofadopljbjfkapdkoienihi), работают только с **неминифицированными** сборками приложений.
:::

## Локатор Vue

:::note

Локатор Vue является экспериментальным и имеет префикс `_`. Функциональность может измениться в будущем.
:::

В локаторе Vue имена компонентов записываются в **kebab-case**.

await page.locator('_vue=book-item').click();

- соответствие по компоненту и **логическому значению**: `_vue=my-button[enabled = false]`
- соответствие по значению свойства **подстроки**: `_vue=[author *= "King"]`
- соответствие по компоненту и **нескольким свойствам**: `_vue=book-item[author *= "king" i][year = 1990]`

Чтобы найти имена элементов Vue в дереве, используйте [Vue DevTools](https://chrome.google.com/webstore/detail/vuejs-devtools/nhdogjmejiglipccpnnnanhbledajbpd?hl=en).

## XPath локатор

:::warning

Мы рекомендуем отдавать предпочтение [видимым пользователю локаторам](./locators.mdx#quick-guide), таким как текст или доступная роль, вместо использования XPath, который связан с реализацией и легко ломается при изменении страницы.
:::

XPath локаторы эквивалентны вызову [`Document.evaluate`](https://developer.mozilla.org/en/docs/Web/API/Document/evaluate).

```js
await page.locator('xpath=//button').click();
```

:::note
Любая строка селектора, начинающаяся с `//` или `..`, считается селектором xpath. Например, Playwright преобразует `'//html/body'` в `'xpath=//html/body'`.
:::

:::note
XPath не проникает в shadow roots.
:::

### Объединение XPath

Оператор пайпа (`|`) может быть использован для указания нескольких селекторов в XPath. Он будет соответствовать всем элементам, которые могут быть выбраны одним из селекторов в этом списке.

```js
// Ожидает либо диалог подтверждения, либо индикатор загрузки.
await page.locator(
    `//span[contains(@class, 'spinner__loading')]|//div[@id='confirmation']`
).waitFor();
```

## Перенаправление метки на элемент управления формы

:::warning

Мы рекомендуем [локализовать по тексту метки](./locators.mdx#locate-by-label) вместо того, чтобы полагаться на перенаправление метки на элемент управления.
:::

Целевые действия ввода в Playwright автоматически различают метки и элементы управления, поэтому вы можете нацелиться на метку, чтобы выполнить действие на связанном элементе управления.

Например, рассмотрим следующую структуру DOM: `<label for="password">Пароль:</label><input id="password" type="password">`. Вы можете нацелиться на метку по ее тексту "Пароль", используя [page.getByText()](/api/class-page.mdx#page-get-by-text). Однако следующие действия будут выполнены на вводе, а не на метке:
- [locator.click()](/api/class-locator.mdx#locator-click) кликнет по метке и автоматически сфокусирует поле ввода;
- [locator.fill()](/api/class-locator.mdx#locator-fill) заполнит поле ввода;
- [locator.inputValue()](/api/class-locator.mdx#locator-input-value) вернет значение поля ввода;
- [locator.selectText()](/api/class-locator.mdx#locator-select-text) выделит текст в поле ввода;
- [locator.setInputFiles()](/api/class-locator.mdx#locator-set-input-files) установит файлы для поля ввода с `type=file`;
- [locator.selectOption()](/api/class-locator.mdx#locator-select-option) выберет опцию из выпадающего списка.

```js
// Заполнить ввод, нацеливаясь на метку.
await page.getByText('Пароль').fill('секрет');
```

Однако другие методы будут нацеливаться на саму метку, например [expect(locator).toHaveText()](/api/class-locatorassertions.mdx#locator-assertions-to-have-text) будет утверждать текстовое содержимое метки, а не поля ввода.

```js
// Заполнить ввод, нацеливаясь на метку.
await expect(page.locator('label')).toHaveText('Пароль');
```

## Устаревший текстовый локатор

:::warning

Мы рекомендуем современный [текстовый локатор](./locators.mdx#get-by-text) вместо этого.
:::

Устаревший текстовый локатор соответствует элементам, которые содержат переданный текст.

```js
await page.locator('text=Log in').click();
```

Устаревший текстовый локатор имеет несколько вариантов:
- `text=Log in` - по умолчанию соответствие нечувствительно к регистру, обрезает пробелы и ищет подстроку. Например, `text=Log` соответствует `<button>Log in</button>`.
  
  ```js
  await page.locator('text=Log in').click();
  ```
  
- `text="Log in"` - текстовое содержимое может быть экранировано одинарными или двойными кавычками для поиска текстового узла с точным содержимым после обрезки пробелов.
  
  Например, `text="Log"` не соответствует `<button>Log in</button>`, потому что `<button>` содержит один текстовый узел `"Log in"`, который не равен `"Log"`. Однако `text="Log"` соответствует `<button> Log <span>in</span></button>`, потому что `<button>` содержит текстовый узел `" Log "`. Этот точный режим подразумевает чувствительное к регистру соответствие, поэтому `text="Download"` не будет соответствовать `<button>download</button>`.
  
  Текст в кавычках следует обычным правилам экранирования, например, используйте `\"` для экранирования двойной кавычки в строке с двойными кавычками: `text="foo\"bar"`.
  
  ```js
  await page.locator('text="Log in"').click();
  ```
  
- `/Log\s*in/i` - текст может быть [регулярным выражением, похожим на JavaScript](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/RegExp), заключенным в символы `/`. Например, `text=/Log\s*in/i` соответствует `<button>Login</button>` и `<button>log IN</button>`.
  
  ```js
  await page.locator('text=/Log\\s*in/i').click();
  ```
  
:::note

Строковые селекторы, начинающиеся и заканчивающиеся кавычкой (либо `"` или `'`), считаются устаревшими текстовыми локаторами. Например, `"Log in"` преобразуется в `text="Log in"` внутренне.
:::

:::note
Сравнение всегда нормализует пробелы. Например, оно превращает несколько пробелов в один, заменяет переносы строк на пробелы и игнорирует начальные и конечные пробелы.
:::

:::note
Элементы ввода типа `button` и `submit` соответствуют по своему `value`, а не по текстовому содержимому. Например, `text=Log in` соответствует `<input type=button value="Log in">`.
:::

## id, data-testid, data-test-id, data-test селекторы

:::warning

Мы рекомендуем [локализовать по тестовому id](./locators.mdx#locate-by-test-id) вместо этого.
:::

Playwright поддерживает сокращенный способ выбора элементов с использованием определенных атрибутов. В настоящее время поддерживаются только следующие атрибуты:
- `id`
- `data-testid`
- `data-test-id`
- `data-test`

```js
// Заполнить ввод с id "username"
await page.locator('id=username').fill('value');

// Кликнуть по элементу с data-test-id "submit"
await page.locator('data-test-id=submit').click();
```

:::note
Селекторы атрибутов не являются CSS-селекторами, поэтому любые специфические для CSS элементы, такие как `:enabled`, не поддерживаются. Для получения дополнительных возможностей используйте правильный [css](#css-locator) селектор, например, `css=[data-test="login"]:enabled`.
:::

## Цепочка селекторов

:::warning

Мы рекомендуем [цеплять локаторы](./locators.mdx#matching-inside-a-locator) вместо этого.
:::

Селекторы, определенные как `engine=body` или в сокращенной форме, могут быть объединены с помощью токена `>>`, например, `selector1 >> selector2 >> selectors3`. Когда селекторы объединены, следующий запрашивается относительно результата предыдущего.

Например,

```txt
css=article >> css=.bar > .baz >> css=span[attr=value]
```

эквивалентен

```js
document
    .querySelector('article')
    .querySelector('.bar > .baz')
    .querySelector('span[attr=value]');
```

Если селектору нужно включать `>>` в теле, он должен быть экранирован внутри строки, чтобы не путаться с разделителем цепочки, например, `text="some >> text"`.

### Промежуточные совпадения

:::warning

Мы рекомендуем [фильтровать по другому локатору](./locators.mdx#filter-by-childdescendant), чтобы находить элементы, которые содержат другие элементы.
:::

По умолчанию объединенные селекторы разрешаются в элемент, запрашиваемый последним селектором. Селектор может быть префиксирован `*`, чтобы захватить элементы, которые запрашиваются промежуточным селектором.

Например, `css=article >> text=Hello` захватывает элемент с текстом `Hello`, а `*css=article >> text=Hello` (обратите внимание на `*`) захватывает элемент `article`, который содержит какой-то элемент с текстом `Hello`.
