## Введение

Playwright может взаимодействовать с HTML элементами ввода, такими как текстовые поля, флажки, радиокнопки, варианты выбора, клики мышью, ввод символов, клавиш и сочетаний клавиш, а также загружать файлы и фокусироваться на элементах.

## Текстовый ввод

Использование [locator.fill()](/api/class-locator.mdx#locator-fill) — это самый простой способ заполнить поля формы. Он фокусируется на элементе и вызывает событие `input` с введенным текстом. Это работает для элементов `<input>`, `<textarea>` и `[contenteditable]`.

```js
// Текстовый ввод
await page.getByRole('textbox').fill('Peter');

// Ввод даты
await page.getByLabel('Birth date').fill('2020-02-02');

// Ввод времени
await page.getByLabel('Appointment time').fill('13:15');

// Ввод локального времени
await page.getByLabel('Local time').fill('2020-03-02T05:15');
```

## Флажки и радиокнопки

Использование [locator.setChecked()](/api/class-locator.mdx#locator-set-checked) — это самый простой способ установить или снять отметку с флажка или радиокнопки. Этот метод можно использовать с элементами `input[type=checkbox]`, `input[type=radio]` и `[role=checkbox]`.

```js
// Установить отметку в флажке
await page.getByLabel('I agree to the terms above').check();

// Проверить состояние отметки
expect(page.getByLabel('Subscribe to newsletter')).toBeChecked();

// Выбрать радиокнопку
await page.getByLabel('XL').check();
```

## Выбор вариантов

Выбирает один или несколько вариантов в элементе `<select>` с помощью [locator.selectOption()](/api/class-locator.mdx#locator-select-option). Вы можете указать значение `value` или `label` для выбора. Можно выбрать несколько вариантов.

```js
// Один выбор, соответствующий значению или метке
await page.getByLabel('Choose a color').selectOption('blue');

// Один выбор, соответствующий метке
await page.getByLabel('Choose a color').selectOption({ label: 'Blue' });

// Несколько выбранных элементов
await page.getByLabel('Choose multiple colors').selectOption(['red', 'green', 'blue']);
```

## Клик мышью

Выполняет простой клик, как это делает человек.

```js
// Обычный клик
await page.getByRole('button').click();

// Двойной клик
await page.getByText('Item').dblclick();

// Правый клик
await page.getByText('Item').click({ button: 'right' });

// Shift + клик
await page.getByText('Item').click({ modifiers: ['Shift'] });

// Ctrl + клик на Windows и Linux
// Meta + клик на macOS
await page.getByText('Item').click({ modifiers: ['ControlOrMeta'] });

// Наведение на элемент
await page.getByText('Item').hover();

// Клик в верхний левый угол
await page.getByText('Item').click({ position: { x: 0, y: 0 } });
```

Внутри, этот и другие методы, связанные с указателем:
- ждут, пока элемент с заданным селектором появится в DOM
- ждут, пока он станет видимым, т.е. не пустым, без `display:none`, без `visibility:hidden`
- ждут, пока он перестанет двигаться, например, пока не завершится CSS-переход
- прокручивают элемент в видимую область
- ждут, пока он получит события указателя в точке действия, например, ждут, пока элемент не будет закрыт другими элементами
- повторяют попытку, если элемент был отсоединен во время любой из вышеуказанных проверок

#### Принудительный клик

Иногда приложения используют нетривиальную логику, при которой наведение на элемент накрывает его другим элементом, который перехватывает клик. Это поведение неотличимо от ошибки, когда элемент закрыт, и клик отправляется в другое место. Если вы знаете, что это происходит, вы можете обойти проверки [доступности](./actionability.mdx) и принудительно выполнить клик:

```js
await page.getByRole('button').click({ force: true });
```

#### Программный клик

Если вы не заинтересованы в тестировании вашего приложения в реальных условиях и хотите смоделировать клик любыми возможными способами, вы можете вызвать поведение [`HTMLElement.click()`](https://developer.mozilla.org/en-US/docs/Web/API/HTMLElement/click), просто отправив событие клика на элемент с помощью [locator.dispatchEvent()](/api/class-locator.mdx#locator-dispatch-event):

```js
await page.getByRole('button').dispatchEvent('click');
```

## Ввод символов

:::caution

Чаще всего вы должны вводить текст с помощью [locator.fill()](/api/class-locator.mdx#locator-fill). См. раздел [Текстовый ввод](#text-input) выше. Вам нужно вводить символы только в том случае, если на странице есть специальная обработка клавиатуры.
:::

Вводите в поле символ за символом, как если бы это делал пользователь с настоящей клавиатурой с помощью [locator.pressSequentially()](/api/class-locator.mdx#locator-press-sequentially).

```js
// Нажимайте клавиши по одной
await page.locator('#area').pressSequentially('Hello World!');
```

Этот метод будет генерировать все необходимые события клавиатуры, с событиями `keydown`, `keyup`, `keypress` на месте. Вы даже можете указать необязательную `delay` между нажатиями клавиш, чтобы смоделировать поведение реального пользователя.

## Клавиши и сочетания клавиш

```js
// Нажать Enter
await page.getByText('Submit').press('Enter');

// Отправить Control+Right
await page.getByRole('textbox').press('Control+ArrowRight');

// Нажать знак $ на клавиатуре
await page.getByRole('textbox').press('$');
```

Метод [locator.press()](/api/class-locator.mdx#locator-press) фокусируется на выбранном элементе и производит одно нажатие клавиши. Он принимает логические имена клавиш, которые передаются в свойстве [keyboardEvent.key](https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/key) событий клавиатуры:

```txt
Backquote, Minus, Equal, Backslash, Backspace, Tab, Delete, Escape,
ArrowDown, End, Enter, Home, Insert, PageDown, PageUp, ArrowRight,
ArrowUp, F1 - F12, Digit0 - Digit9, KeyA - KeyZ и т.д.
```

- Вы также можете указать один символ, который хотите получить, например, `"a"` или `"#"`.
- Поддерживаются следующие модификаторы: `Shift, Control, Alt, Meta`.

Простая версия производит один символ. Этот символ чувствителен к регистру, поэтому `"a"` и `"A"` будут давать разные результаты.

```js
// <input id=name>
await page.locator('#name').press('Shift+A');

// <input id=name>
await page.locator('#name').press('Shift+ArrowLeft');
```

Сочетания клавиш, такие как `"Control+o"` или `"Control+Shift+T"`, также поддерживаются. Когда указано с модификатором, модификатор удерживается нажатым, пока нажимается последующая клавиша.

Обратите внимание, что вам все равно нужно указывать заглавную `A` в `Shift-A`, чтобы получить заглавный символ. `Shift-a` производит строчную букву, как если бы у вас был переключен `CapsLock`.

## Загрузка файлов

Вы можете выбрать файлы для загрузки с помощью метода [locator.setInputFiles()](/api/class-locator.mdx#locator-set-input-files). Он ожидает, что первый аргумент указывает на [элемент input](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input) с типом `"file"`. В массив можно передать несколько файлов. Если некоторые из путей к файлам относительные, они разрешаются относительно текущего рабочего каталога. Пустой массив очищает выбранные файлы.

```js
// Выбрать один файл
await page.getByLabel('Upload file').setInputFiles(path.join(__dirname, 'myfile.pdf'));

// Выбрать несколько файлов
await page.getByLabel('Upload files').setInputFiles([
  path.join(__dirname, 'file1.txt'),
  path.join(__dirname, 'file2.txt'),
]);

// Выбрать директорию
await page.getByLabel('Upload directory').setInputFiles(path.join(__dirname, 'mydir'));

// Удалить все выбранные файлы
await page.getByLabel('Upload file').setInputFiles([]);

// Загрузить буфер из памяти
await page.getByLabel('Upload file').setInputFiles({
  name: 'file.txt',
  mimeType: 'text/plain',
  buffer: Buffer.from('this is test')
});
```

Если у вас нет элемента input под рукой (он создается динамически), вы можете обработать событие [page.on('filechooser')](/api/class-page.mdx#page-event-file-chooser) или использовать соответствующий метод ожидания перед вашим действием:

```js
// Начните ожидать выбора файла перед кликом. Обратите внимание, что нет await.
const fileChooserPromise = page.waitForEvent('filechooser');
await page.getByLabel('Upload file').click();
const fileChooser = await fileChooserPromise;
await fileChooser.setFiles(path.join(__dirname, 'myfile.pdf'));
```

## Фокус на элементе

Для динамических страниц, которые обрабатывают события фокуса, вы можете установить фокус на данный элемент с помощью [locator.focus()](/api/class-locator.mdx#locator-focus).

```js
await page.getByLabel('Password').focus();
```

## Перетаскивание и сброс

Вы можете выполнить операцию перетаскивания и сброса с помощью [locator.dragTo()](/api/class-locator.mdx#locator-drag-to). Этот метод:
- Наводит указатель на элемент, который будет перетаскиваться.
- Нажимает левую кнопку мыши.
- Перемещает указатель на элемент, который будет принимать сброс.
- Отпускает левую кнопку мыши.

```js
await page.locator('#item-to-be-dragged').dragTo(page.locator('#item-to-drop-at'));
```

### Ручное перетаскивание

Если вам нужен точный контроль над операцией перетаскивания, используйте более низкоуровневые методы, такие как [locator.hover()](/api/class-locator.mdx#locator-hover), [mouse.down()](/api/class-mouse.mdx#mouse-down), [mouse.move()](/api/class-mouse.mdx#mouse-move) и [mouse.up()](/api/class-mouse.mdx#mouse-up).

```js
await page.locator('#item-to-be-dragged').hover();
await page.mouse.down();
await page.locator('#item-to-drop-at').hover();
await page.mouse.up();
```

:::note
Если ваша страница зависит от события `dragover`, которое должно быть отправлено, вам нужно как минимум два перемещения мыши, чтобы вызвать его во всех браузерах. Чтобы надежно выполнить второе перемещение мыши, повторите ваше [mouse.move()](/api/class-mouse.mdx#mouse-move) или [locator.hover()](/api/class-locator.mdx#locator-hover) дважды. Последовательность операций будет следующей: наведите указатель на элемент перетаскивания, нажмите кнопку мыши, наведите указатель на элемент сброса, наведите указатель на элемент сброса второй раз, отпустите кнопку мыши.
:::

## Прокрутка

Чаще всего Playwright будет автоматически прокручивать для вас перед выполнением любых действий. Поэтому вам не нужно прокручивать явно.

```js
// Автоматически прокручивает, чтобы кнопка была видна
await page.getByRole('button').click();
```

Однако в редких случаях вам может понадобиться прокрутить вручную. Например, вы можете захотеть заставить "бесконечный список" загрузить больше элементов или позиционировать страницу для конкретного скриншота. В таком случае самым надежным способом будет найти элемент, который вы хотите сделать видимым внизу, и прокрутить его в видимую область.

```js
// Прокрутить нижний колонтитул в видимую область, заставляя "бесконечный список" загружать больше контента
await page.getByText('Footer text').scrollIntoViewIfNeeded();
```

Если вы хотите более точно контролировать прокрутку, используйте [mouse.wheel()](/api/class-mouse.mdx#mouse-wheel) или [locator.evaluate()](/api/class-locator.mdx#locator-evaluate):

```js
// Позиционируйте указатель мыши и прокручивайте с помощью колеса мыши
await page.getByTestId('scrolling-container').hover();
await page.mouse.wheel(0, 10);

// Либо программно прокрутите конкретный элемент
await page.getByTestId('scrolling-container').evaluate(e => e.scrollTop += 100);
```
